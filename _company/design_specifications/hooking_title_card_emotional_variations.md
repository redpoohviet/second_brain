# 🎨 Hooking Title Card: 감성별 변주 규칙 및 최종 기술 사양 (V4.0)

## 💡 개요 및 목표
본 문서는 'Hooking Title Card' 모듈이 단순히 정보를 전달하는 것을 넘어, 시청자의 심리적 반응(Pain Point)을 극대화하고 다음 내용에 대한 호기심(Curiosity Gap)을 유도하기 위한 **감성 기반의 인터랙티브 명세서**입니다.

모든 요소는 CSS 변수(`var(--color-...)`, `var(--size-...)`)와 시간 경과에 따른 애니메이션으로 정의되어야 하며, 개발팀은 이 사양을 바탕으로 코딩 모듈을 완성해야 합니다.

## 🛠️ 공통 컴포넌트 및 시스템 요구사항
1. **배경:** 뉴트럴 어스톤 배경 위에 미세한 노이즈 레이어 유지 (`opacity: 0.1`).
2. **타입스크립트 변수:** 모든 크기와 색상은 반드시 CSS 변수를 사용합니다. (예: `var(--size-h1)`, `var(--color-text-dark)`)
3. **애니메이션 로직:** 모든 진입 및 전환(Transition)은 부드러운 비선형 애니메이션(`ease-out` 또는 `cubic-bezier(...)`)을 사용합니다.

## 🎭 변주 유형 1: [충격/경고] (Shock / Alarm Hooking)
* **감성적 의도:** 시청자에게 '지금 보고 있는 정보가 상식을 벗어났다'는 느낌을 주며, 경계심과 강한 주의를 유발합니다. 가장 높은 주목도를 요구할 때 사용됩니다.
* **사용 상황:** "이 방법은 아무도 알려주지 않았습니다." / "당신이 틀렸을 수도 있습니다."와 같은 논쟁적/충격적인 주제의 도입부.

### ⚙️ 기술 사양 (Tech Spec)
| 요소 | 속성 | 값 및 CSS 정의 | 애니메이션 로직 |
| :--- | :--- | :--- | :--- |
| **전체 배경** | 색상 변화 | `var(--color-bg-primary)` 유지. 다만, 노이즈 레이어 위에 얇은 **빨간색 경고 오버레이** (`rgba(255, 0, 0, 0.1)`)가 미세하게 깜빡이는(`animation: blink 1s infinite step-start;`) 효과 적용. | `background-attachment: fixed`으로 배경이 움직이지 않도록 고정. |
| **메인 타이틀** | 색상 및 스타일 | `#D42B3A` (강렬한 버건디 레드). 모든 글자가 대문자(UPPERCASE) 처리. 폰트 가중치 극대화 (`font-weight: 900`). | **[진입 애니메이션]**: `opacity: 0`에서 시작하여, 짧은 시간(`transition-duration: 0.2s`)에 걸쳐 **강하게 '팝업'** 하듯 나타납니다. (Scale Up + Fade In) |
| **보조 자막/근거** | 시각화 | 시스템 경고창(System Alert Box) 형태의 디자인을 채택합니다. 사각형 테두리를 사용하고, 왼쪽 상단에 `[!! WARNING !!]`과 같은 플래시 텍스트를 추가합니다. | **[진입 애니메이션]**: 메인 타이틀 등장 직후, 좌측에서 오른쪽으로 빠르게 슬라이드 인(Slide-In) 하며 나타납니다. (Timing: $0.2s$ 지연). |
| **핵심 로직** | 사운드/시각 동기화 | 텍스트가 화면에 진입하는 순간, 낮은 주파수의 경고음(`BWOOOM`)과 함께 배경의 빨간 오버레이 깜빡임이 최대치로 증가합니다. (사운드 API 연동 필수) | N/A |

## 🎭 변주 유형 2: [호기심 유발] (Curiosity Gap Hooking)
* **감성적 의도:** 시청자에게 '어떻게?', '왜?'라는 질문을 던지며, 답을 찾고 싶게 만듭니다. 직접적인 충격보다 은근한 궁금증을 통해 몰입도를 높입니다.
* **사용 상황:** "만약 이 과정을 거치면 어떻게 될까요?" / "당신은 이 사실을 모르셨을 겁니다."와 같은 추론적, 학습 기반의 주제 도입부.

### ⚙️ 기술 사양 (Tech Spec)
| 요소 | 속성 | 값 및 CSS 정의 | 애니메이션 로직 |
| :--- | :--- | :--- | :--- |
| **전체 배경** | 색상 변화 | `var(--color-bg-secondary)` (밝은 크림색/종이 질감). 아날로그 필름 노이즈는 유지하되, 미세한 빛 번짐(Soft Vignette) 효과를 추가합니다. | 부드럽고 지속적인 Parallax Scrolling 느낌을 주어 깊이감을 부여합니다. |
| **메인 타이틀** | 색상 및 스타일 | `var(--color-text-dark)` (차콜 그레이). 폰트는 가독성 높은 산세리프체를 사용하되, 마치 손으로 메모한 듯한 느낌의 서체(Semi-Handwritten)를 적용합니다. | **[진입 애니메이션]**: 타이틀이 화면 중앙에서 아주 느리고 부드럽게 '떠오르는' 효과(`transition-duration: 0.8s; ease-out`)를 구현합니다. (Fade In + Gentle Float Up). |
| **보조 자막/근거** | 시각화 | 질문 형식의 구문("~하는 원리?", "과연 가능할까요?")을 사용하며, 타이틀 아래에 출처나 '궁금한 사람들을 위한 노트' 같은 모듈 형태로 배치합니다. | **[진입 애니메이션]**: 마치 누군가 손글씨로 메모하듯, 텍스트가 한 글자씩 순차적으로 나타납니다 (Typewriter Effect). 시청자가 읽는 속도에 맞춰 타이밍을 조절해야 합니다. |
| **핵심 로직** | 상호작용 유도 | "더 알아보기" 또는 물음표 아이콘(`?`) 클릭 시, 화면의 좌측이나 우측에서 관련 정보를 담은 팝업/모듈이 부드럽게 펼쳐지며(Accordion Effect) 다음 내용에 대한 예고를 합니다. (클릭 이벤트 리스너 필수). | N/A |

## 🎭 변주 유형 3: [권위 확립] (Authority / Trust Hooking)
* **감성적 의도:** "이것은 공신력 있는 정보다", "전문가들이 입증한 방법이다"라는 느낌을 주어, 시청자에게 안정적인 신뢰감을 심어줍니다.
* **사용 상황:** 구체적인 데이터나 시스템 프로세스(OEA 등)를 설명하며 전문성을 강조할 때 사용됩니다.

### ⚙️ 기술 사양 (Tech Spec)
| 요소 | 속성 | 값 및 CSS 정의 | 애니메이션 로직 |
| :--- | :--- | :--- | :--- |
| **전체 배경** | 색상 변화 | `var(--color-bg-primary)`를 유지하되, 전체적으로 깨끗하고 정돈된 느낌의 톤을 가져갑니다. (필름 노이즈는 최소화). 핵심 정보가 담긴 박스 영역에만 대비되는 밝은 배경색(`var(--color-bg-secondary)`)을 사용합니다. | 전반적인 움직임 없이, **정보를 배치하는 그리드 시스템** 자체가 안정감을 주도록 디자인합니다. (Minimal Movement). |
| **메인 타이틀** | 색상 및 스타일 | 짙고 신뢰감 있는 네이비 계열(`var(--color-text-dark)`와 유사하나 약간 더 깊은 블루톤)을 사용합니다. '데이터', '분석', '원리' 같은 단어에 강조점을 줍니다. | **[진입 애니메이션]**: 타이틀과 부제목이 좌우 대칭으로 동시에 나타나며, 마치 보고서가 펼쳐지는 듯한 정적인 느낌(Steady Reveal)을 극대화합니다. (Ease: `ease-in-out`). |
| **보조 자막/근거** | 시각화 | 반드시 데이터 그래프, 차트, 프로세스 플로우 다이어그램 등 '수치'와 '구조'를 포함해야 합니다. 각 수치는 출처(Source)와 함께 명시되어야 하며, 모노 스페이스체(`Courier New`)를 사용하여 전문성을 부여합니다. | **[진입 애니메이션]**: 차트의 막대나 라인 그래프가 마치 데이터 로딩처럼 0부터 시작하여 목표 값까지 부드럽게 채워지는(Animated Line Growth) 효과를 적용해야 합니다. (Timing: $1s$에 걸쳐 점진적). |
| **핵심 로직** | 전문성 강조 | 모든 정보 블록에는 출처 또는 참고 논문 번호 등을 명시하고, 이를 'Verification Check'와 같은 시각적 장치로 표현하여 신뢰도를 높입니다. (데이터 기반의 애니메이션 필수). | N/A |
