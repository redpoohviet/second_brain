### 🛠️ 🎨 수익화 퍼널 통합 디자인 시스템 (V1.0) Tech Spec

**목표:** 현빈님이 정의한 3단계 퍼널(Tier A, B, C)에 맞는 모든 비주얼 컴포넌트의 개발 표준을 확립합니다.
**원칙:** 모든 값은 CSS 변수를 사용하며, 단순 이미지가 아닌 **상태 변화(State Change)**를 염두에 둔 인터랙티브 요소로 정의됩니다.

#### 1. 공통 디자인 시스템 기반 (Global Variables)

| 영역 | Variable Name | 값/설명 | 근거 |
|------|---------------|--------|-----|
| 배경색 | `--color-bg-primary` | `#DDC2B9` (뉴트럴 어스톤) | Designer 검증된 지식 |
| 텍스트색 | `--color-text-dark` | `#3A3530` (짙은 차콜 그레이) | Designer 검증된 지식 |
| 보조 배경 | `--color-bg-secondary` | `#F5EFEA` (밝은 크림색) | Designer 검증된 지식 |
| 강조색(Accent) | `--color-accent-success` | `#7B8E4C` (차분한 녹색 계열, '통제권' 상징) | *[추측]* - 수익화 성공/신뢰감 유도 색상 |
| 제목폰트 | `--font-family-hook` | `Brush Script`, 캘리그래피체 | Designer 검증된 지식 |
| 본문폰트 | `--font-family-body` | `Pretendard` 또는 `Noto Sans KR` (산세리프) | Designer 검증된 지식 |

#### 2. 🎬 비디오 썸네일 컴포넌트 Tech Spec (Video Thumbnail Component)

**목표:** 시청자의 주의를 끌어 클릭을 유도하는 후킹 자막 및 핵심 메시지 전달에 최적화됩니다.
**규격:** 16:9 비율, 최소 해상도 1280px x 720px (모바일 환경 대비 여유 공간 확보).

| 요소 | 기능/내용 | 크기/배치 | 애니메이션 로직 (Interaction) | 근거 |
|------|-------------|-----------|-------------------------------|-----|
| **HOOK** | 가장 큰 문구. Pain Point를 직접 언급. ("이 방법, 왜 이제 알았을까요?") | 상단 30% 영역 중앙 배치. `--font-family-hook` 사용. | **Fade In (Duration: 0.2s)** + **Typewriter Effect** (타자 효과). 시청자가 영상을 보는 순간 즉각적 주목 유도. | Designer 검증된 지식 / Writer 메모리 |
| **PROBLEM ICON** | 문제 상황을 상징하는 아이콘/이미지 (예: 복잡한 플로우차트, ? 표시) | 좌측 25% 영역에 배치. 배경은 `var(--color-bg-secondary)` 사용. | 영상 시작과 동시에 **Zoom In (Scale: 1.0 → 1.1)** 효과를 주어 시각적 불안감을 증폭시킴. | Designer 개인 메모리 / Researcher |
| **SOLUTION/CTA** | 핵심 솔루션 키워드 및 CTA 유도 문구. ("OEA 시스템화", "통제권 확보") | 하단 25% 영역에 배치. `--color-accent-success` 강조색 사용. | 영상 중간(T=7s)경, **Slide Up (Duration: 0.4s)** 애니메이션으로 등장하며 해결책을 제시함. | Designer 검증된 지식 / 현빈 전략 |

#### 3. 💻 랜딩 페이지 컴포넌트 Tech Spec (Landing Page Component)

**목표:** 구매 퍼널의 각 단계(Tier A, B, C)에 맞는 설득력 높은 CTA 및 상세 설명 섹션 구현.
**규격:** 웹 표준 반응형 디자인 (`max-width: 1200px`).

##### A. Tier A (진입/Hooking): '문제 인식' 강조 섹션
*   **헤드라인:** "혹시 지금도 이런 방식으로 시간과 돈을 낭비하고 계신가요?"
*   **레이아웃:** **좌우 분할 구조** (`display: flex;`) 사용.
    *   **Left Side (Pain Point):** 문제 상황의 구체적인 예시 목록을 체크리스트 형태로 나열 (불만족 요소 강조). 배경에 미세 노이즈 오버레이 적용 (`opacity: 0.1`). [근거: Designer 검증된 지식]
    *   **Right Side (Solution Teaser):** OEA 프로세스의 가장 간단한 개념도를 제시하고, CTA 버튼을 배치.
        *   **CTA Button:** `#27 USD` 강조 색상 사용. **Hover Effect**: 마우스 오버 시 배경이 `var(--color-accent-success)`로 변하며 미세하게 크기 증가 (`transform: scale(1.05)`).

##### B. Tier B (신뢰/Authority): '시스템' 증명 섹션
*   **핵심:** 기술적 전문성(System)을 시각화하여 신뢰를 구축하는 것이 목표입니다.
*   **레이아웃:** **탭 인터랙션 시스템** (`<details>` 태그 또는 JavaScript 탭).
    *   3개의 탭: [관찰] / [추출] / [자동화].
    *   **상호작용 로직:** 사용자가 탭을 클릭하면, 해당 단계의 **기술 플로우차트(Tech Flowchart)**가 부드럽게 나타나며 설명을 보여줍니다. (Transition Duration: 0.6s). 이는 단순히 글을 읽는 것이 아니라 '시스템 원리를 이해'하게 만듭니다. [근거: Designer 개인 메모리]
    *   **CTA Button:** `#49-$99` 강조 색상 사용. **Click Animation**: 클릭 시, 버튼 배경이 짧게 깜빡이며(Blink Effect) "결제 과정에 대한 안내가 곧 진행됩니다"라는 텍스트를 오버레이합니다.

##### C. Tier C (최종 전환/Closing): '완벽한 통제' 제시 섹션
*   **핵심:** 모든 것을 한 번에 해결한다는 느낌을 주며, AOV 극대화를 목표로 합니다.
*   **레이아웃:** **비교표(Pricing Comparison Table)** + 강력한 최종 CTA.
    *   **테이블 디자인:** 3열 구조 (A vs B vs C). Tier C 열만 `--color-accent-success` 배경으로 강조 처리합니다.
    *   **최종 CTA Button:** `#150+ USD`. **Interaction**: 가장 크고, 시각적으로 압도적이며, 마우스 오버 시 '통제권 획득'을 상징하는 작은 애니메이션(예: 데이터가 채워지는 효과)이 실행되어야 합니다.
