## 📑 1. 개요 및 목표
본 문서는 'Chaos$o$Order' 테마 기반의 모든 핵심 시각적 자산(썸네일, 인트로/아웃트로)이 실제 콘텐츠 플랫폼에 안정적으로 배포되기 위한 최종 기술 사양서이자 체크리스트입니다. 단순한 디자인 가이드가 아닌, 코다리가 개발 및 자동화 파이프라인 구축에 필요한 **'구현 가능한 상세 명세(Actionable Tech Spec)'**를 제공합니다.

## 🎨 2. 비주얼 시스템 변수 (CSS Variables)
모든 에셋은 다음 CSS 변수를 기반으로 크기와 색상이 정의되어야 합니다. [근거: Designer 검증된 지식]

| 변수명 | 값 | 설명 | 용도 |
| :--- | :--- | :--- | :--- |
| `--color-bg-primary` | `#DDC2B9` | 뉴트럴 어스톤 (Primary BG) | 배경색, 주요 섹션 배경 |
| `--color-text-dark` | `#3A3530` | 짙은 차콜 그레이 | 본문/제목 텍스트 색상 |
| `--color-secondary-bg` | `#F5EFEA` | 밝은 크림색 (Secondary BG) | 정보 블록, 대비 영역 배경 |
| `--font-family-hook` | `'Brush Script', cursive` | 손글씨 느낌 | 핵심 슬로건, 후킹 문구 |
| `--font-family-body` | `Pretendard, 'Noto Sans KR'` | 모던 산세리프 | 본문 설명, 시스템 프로세스 |

## 📐 3. 에셋별 상세 사양 및 애니메이션 가이드라인

### A. 메인 썸네일 (Thumbnail Assets)
**목표:** 사용자의 Pain Point를 즉각적으로 자극하고(Hook), '해결책'을 기대하게 만드는 매트릭스 기반의 시각적 충격 유발. [근거: 지난 의사결정 로그]

| 항목 | 사양 | 상세 기술 명세 |
| :--- | :--- | :--- |
| **최종 크기 (YouTube)** | 1280px x 720px (16:9) | `--size-w`: 1280px, `--size-h`: 720px |
| **최종 크기 (Instagram)** | 1080px x 1350px (4:5 또는 9:16 활용) | *플랫폼별 레이아웃 분할 필수* |
| **핵심 요소 배치** | 좌측 Pain Point Trigger / 우측 Solution/OEA 흐름도 | `--coord-left`: 0% ~ 35%, `--coord-right`: 35% ~ 100% |
| **애니메이션 (Aha! Moment)** | **데이터 로딩 효과:** 초기에는 노이즈 레이어만 보이다가, 'OEA' 키워드가 스캐닝(Scanning) 애니메이션과 함께 차례로 등장 (`transition-duration: 0.3s;` / `animation: scanIn linear forwards;`). [근거: Designer 개인 메모리] |

### B. 인트로/아웃트로 카드 (Transition Assets)
**목표:** 콘텐츠 전환 시 발생하는 지루함을 없애고, 채널의 정체성(Tech Spec 느낌)을 유지하며 다음 내용으로 자연스럽게 몰입 유도. [근거: Designer 검증된 지식]

| 항목 | 사양 | 상세 기술 명세 |
| :--- | :--- | :--- |
| **사이즈** | 1080px x 1920px (모바일 최적화) | `--size-w`: 1080px, `--size-h`: 1920px |
| **컴포넌트 1: 로딩 카드** | 'DATA LOADING...' 문구와 빈티지 필름 노이즈 오버레이. | **애니메이션:** 화면 중앙에서 위아래로 스크롤되는 가상 데이터 스트림 (CSS Keyframe Animation). `opacity`를 0.8부터 0.2까지 변환하며 신뢰감 부여. |
| **컴포넌트 2: CTA 카드** | "다음 영상에서는 [주제]에 대한 OEA 프로세스를 공개합니다." | **애니메이션:** 배경색이 `--color-bg-secondary`에서 `--color-text-dark`로 부드럽게 전환되며, CTA 버튼만 미세하게 Pulse 효과를 줌. `transition-timing: ease-out`. [근거: Designer 개인 메모리] |

## ⚙️ 4. 개발 및 배포 체크리스트 (Code Readiness Check)
코다리는 다음 항목들을 필수적으로 검증해야 합니다.

1. **[필수] 반응형 구현:** 모든 컴포넌트는 `vw`와 `%` 단위를 사용하여 디바이스 크기에 관계없이 깨지지 않아야 합니다.
2. **[필수] 인터랙션 정의:** 단순한 비디오 클립으로 끝내지 않고, 영상 내에서 '마우스 오버 시 다음 정보가 팝업되는' 상호작용 로직이 가능하도록 JS/API 이벤트 리스너를 설계해야 합니다. (예: OEA 단계 박스를 클릭하면 해당 프로세스의 상세 설명이 플로우차트 옆에 펼쳐지는 모달).
3. **[필수] 파일 시스템 통합:** 썸네일 에셋은 `assets/thumbnails/` 폴더에, 인트로/아웃트로 시퀀스는 `assets/transitions/` 폴더에 표준 포맷(WebP for 썸네일, MP4/WEBM for 트랜지션)으로 저장되어야 합니다.
