# 🎨 Designer — PNG 형태로의 요청에 대해, 왜 인터랙티브 프로토타입(Figma/Adobe XD 등)이 필수적인지 원칙적으로 설명하는 가이드라인을 만드십시오. 대신, 디자이너가 이해하기 쉽도록 '핵심 단계별 모바일 목업 시퀀스' 형태를 제안합니다. 이 시퀀스는 전체 애니메이션 플로우 중 가장 중요한 3~5개의 키프레임(Keyframe)만을 추출하여, 각 프레임마다 상세한 기술 사양서(Tech Spec)와 함께 보여주는 방식으로 재정의해야 합니다.

🎨 Designer: 작업 시작합니다. 사용자님의 반복적인 요청을 이해했습니다. "PNG 파일로 결과물을 보고 싶다"는 것은 최종 산출물의 **'결과물(Outcome)'**만 확인하고 싶다는 의미입니다. 하지만 우리 프로젝트의 핵심 가치는 '정보가 흘러가는 방식(Flow)', 즉 **'경험(Experience)'** 그 자체에 있습니다.

이 원칙적인 차이를 설명하는 동시에, 사용자님께서 시각적으로 이해하기 쉽도록 최적화된 새로운 프레젠테이션 방식을 제안하겠습니다.

---

## 📘 섹션 1: 왜 정지 이미지(PNG)만으로는 부족한가? (원칙 설명)

우리가 만들고자 하는 것은 단순한 **'그림(Illustration)'**이 아니라, 사용자의 심리를 건드리고 행동을 유발하는 **'인터랙티브 시스템 컴포넌트'**입니다. [근거: Designer 개인 메모리]

| 요소 | 정지 이미지 (PNG)의 한계 | 인터랙티브 프로토타입의 가치 |
| :--- | :--- | :--- |
| **핵심 가치** | 특정 시점의 *스냅샷*만 제공. | 시간 경과에 따른 *변화(Change)*를 정의함. |
| **정보 전달** | 정적인 레이아웃을 보여줄 뿐, 정보가 어디서 오고 사라지는지 설명 불가. | 데이터 로딩 애니메이션, 마우스 오버/클릭 반응 등 **상태 변화(State Change)**의 흐름을 구현 가능. [근거: Designer 개인 메모리] |
| **개발 관점** | 개발자가 동작 여부를 수작업으로 추측해야 함. (추가 질문 발생) | 모든 요소의 크기, 타이밍(Timing), 애니메이션 키프레임이 코드로 정의되어 있어 개발 시간이 단축됨. [근거: Designer 개인 메모리] |
| **사용자 경험** | 시청자가 '흐름'을 이해하기 어려움. (단순한 광고 이미지로 인식) | 사용자의 Pain Point를 *강제적으로 상기*시키고, 해결책(Solution)으로 자연스럽게 유도하는 감성적 설계가 가능함. [근거: 지난 의사결정 로그] |

**결론:** 우리의 산출물은 '무엇이 보이는지'보다 **'어떻게 변화하며 보여지는지'**를 기술하는 것이 핵심입니다. PNG는 이 복잡한 과정을 한 장의 그림으로 압축할 수 없습니다.

## 🎬 섹션 2: 새로운 제안 - [키 단계별 모바일 목업 시퀀스] 가이드라인

PNG 요청을 만족시키면서도 기술적 정확도를 유지하기 위해, **전체 애니메이션 플로우를 3~5개의 가장 중요한 'Aha! Moment'로 분할하여 보여주는 방식**을 채택합니다. [근거: Designer 개인 메모리]

이는 마치 영화의 전체 러닝타임을 보여주는 대신, 클라이맥스(Climax)가 되는 핵심 장면 몇 개만 편집해서 보여주는 것과 같습니다.

### ✨ 산출물 구조 (Output Structure)

| 항목 | 설명 (What to Show) | 포함할 기술 사양서 (Tech Spec Focus) |
| :--- | :--- | :--- |
| **1. 키프레임 제목** | 이 단계에서 사용자가 느끼는 감정적 전환점 명시. (예: "충격적인 비효율성 인지") | N/A |
| **2. 모바일 목업 시퀀스** | 해당 순간의 화면 레이아웃과 핵심 요소들의 배치(Mockup Image). (가장 직관적인 부분) | CSS 변수 기반 좌표 정의, 크기 (`var(--size-h1)`), 폰트 정보. [근거: Designer 검증된 지식] |
| **3. 애니메이션 흐름** | 이 화면에 도달하기까지의 동작(Transition)을 설명합니다. (예: "화면 좌측에서 0.5초간 슬라이드 인") | **타이밍 가이드:** `transition-duration: 0.5s;` / **애니메이션 함수:** `ease-out`. [근거: Designer 개인 메모리] |
| **4. 상호작용 로직** | 이 순간에 어떤 코드가 작동해야 하는지 정의합니다. (가장 중요한 부분) | **JS/API 이벤트 리스너 명세서:** `onClick={handle_data_load}` 또는 `onScroll={check_scroll_depth}`. [근거: Designer 개인 메모리] |
