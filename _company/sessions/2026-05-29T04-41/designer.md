### 🎬 초기 비주얼 컨셉 명세서: 후킹 타이틀 카드 (Short-Form Video)

이 명세서는 Writer가 제시한 주제에 관계없이, 채널의 모든 쇼츠 콘텐츠 도입부(Hook)에 일관성 있게 적용될 표준화된 '인터랙티브 시스템 컴포넌트'입니다. 정적인 이미지가 아닌, **동작 타이밍과 레이아웃 변화**를 중심으로 기술합니다.

#### 1. 목표 및 컨셉 (Goal & Concept)
*   **목표:** 시청자의 스크롤을 멈추고(Stop Scroll), 콘텐츠의 주제에 대해 강한 호기심 또는 공감(Pain Point Trigger)을 유발하여 다음 정보 흐름으로 자연스럽게 진입시키는 것. [근거: Designer 개인 메모리]
*   **핵심 컨셉:** **'숨겨진 비밀 폭로(The Reveal)'**. 평범한 상황에 대한 문제 제기에서 시작하여, 해결책이 명확히 제시되는 구조를 모바일 화면 비율에 최적화합니다.

#### 2. 기술 사양 (Technical Specification)
*   **포맷:** 9:16 (모바일 세로형). 최소 해상도: 1080px x 1920px.
*   **배경:** `--color-bg-primary` (`#DDC2B9`)를 기반으로, 빈티지 필름 질감 레이어(`opacity: 0.1`)가 전체적으로 깔립니다. (CSS Gradient 또는 Overlay 사용) [근거: Designer 검증된 지식]
*   **애니메이션:** 모든 요소는 '진입(In)'과 '체류(Stay)'의 두 가지 상태를 가집니다.

#### 3. 레이아웃 및 컴포넌트 상세 명세 (Layout & Component Breakdown)

| 요소 | 역할 / 내용 | 스타일 사양 | 애니메이션 로직 (Timing) |
| :--- | :--- | :--- | :--- |
| **A. 메인 후킹 타이틀** | 가장 자극적이고 질문형의 문구 ("왜 이걸 몰랐을까요?", "시간 낭비 STOP!") | `--color-text-dark` (`#3A3530`). `Brush Script` 또는 유사 캘리그래피체 (H1 크기). **최대 7자 이내.** | **(Entrance):** 화면 상단 중앙에서 좌우로 빠르게 '타이핑 효과'처럼 페이드 인하며 등장. (Duration: 0.4s, Easing: ease-out) |
| **B. 서브 트리거 문구** | 메인 타이틀을 구체화하는 질문/현상 제시 ("매일 이렇게 시간을 보내고 계신가요?") | `--color-text-dark`. `Pretendard` (Body 크기). 2줄 구성. | **(Entrance):** A가 등장한 후 0.3초 지연되어, 하단 중앙에서 '부드러운 페이드 인'으로 나타남. (Duration: 0.6s) |
| **C. 시각적 강조 박스 (OEA Trigger)** | 주제와 관련된 핵심 키워드를 담은 사각형 모듈. (예: `[시스템 오류]`, `[AI 자동화]`) | `--color-bg-secondary` (`#F5EFEA`). 테두리만 얇게 설정. `Courier New` 폰트. | **(Entrance):** B 문구의 특정 단어 옆에서 '점진적으로 확장되는 애니메이션'으로 나타남. (Duration: 0.5s) |
| **D. CTA 유도 가이드** | 다음 영상에 대한 기대감을 높이는 짧은 지시문 ("풀 스토리는 본편을 확인하세요.") | `--color-text-dark`. `Pretendard` (Small 크기). 화면 최하단 고정. | **(Stay):** 콘텐츠 시작 시점부터 끝까지 지속적으로 얇고 은은하게 깜빡이는 애니메이션(`opacity: 0.8` -> `opacity: 1` 반복)을 유지하여 주의를 환기시킵니다. |

#### 4. 개발 지침 (Implementation Directive)
*   **CSS 변수 의무 사용:** 모든 색상, 크기는 반드시 `var(--color-...)`로 참조해야 합니다. [근거: Designer 검증된 지식]
*   **애니메이션 관리:** 애니메이션 타이밍은 **JavaScript의 State Change 이벤트 리스너**를 통해 제어되어야 하며, 단순한 CSS 트랜지션으로 끝나서는 안 됩니다. (예: `onSceneLoad()` 함수 호출 시 A, B가 순차적으로 활성화됨). [근거: Designer 개인 메모리]
