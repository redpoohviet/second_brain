### 📄 산출물: 랜딩 페이지 통합 인터랙티브 기술 사양서 (Master Interactive Tech Spec v1.0)

이 문서는 판매 페이지 초안(UX/UI)을 기반으로, **개발팀에게 전달할 최종 코딩 지침**이며, 정적인 PNG 이미지가 아닌 '상호작용하는 시스템의 청사진'입니다.

#### 1. 목표 및 컨셉 정의
*   **최종 목표:** 방문자에게 Pain Point를 상기시키고(Hooking), 우리의 Solution(OEA)이 유일한 해결책임을 인지시켜, $27 결제 퍼널로 자연스럽게 진입시키는 것.
*   **핵심 비주얼 컨셉:** 뉴트럴 어스톤 바탕 + 빈티지 필름 질감 (전체적인 아날로그적 신뢰감).
*   **기술 구현 원칙:** 모든 색상과 크기는 CSS 변수(`var(--primary-color)`, `var(--font-size-h1)`)로 관리하며, 정보 노출은 애니메이션을 통해 단계적으로 진행되어야 합니다. [근거: Designer 검증된 지식]

#### 2. 전역 스타일 및 변수 정의 (Global CSS Variables)
모든 요소에 일관성을 부여하기 위해 다음과 같은 CSS 변수를 최우선으로 사용합니다.

| 변수명 | 용도 | 값 (HEX Code) | 역할 | 근거 |
| :--- | :--- | :--- | :--- | :--- |
| `--color-bg-primary` | 메인 배경색 (전체 영역) | `#DDC2B9` (뉴트럴 어스톤) | 안정적, 아날로그 감성 유지. | [근거: Designer 검증된 지식] |
| `--color-text-dark` | 본문/제목 텍스트 | `#3A3530` (짙은 차콜 그레이) | 높은 가독성 확보. | [근거: Designer 검증된 지식] |
| `--color-secondary-bg` | 정보 블록, 강조 박스 배경 | `#F5EFEA` (밝은 크림색) | 대비를 통한 핵심 정보 분리. | [근거: Designer 검증된 지식] |
| `--font-family-headline` | 제목/Hooking | `Brush Script`, 캘리그라피체 계열 | 감성적 연결 유도. | [근거: Designer 검증된 지식] |
| `--font-family-body` | 본문/시스템 설명 | `Pretendard` 또는 `Noto Sans KR` | 명확하고 시스템적인 정보 전달. | [근거: Designer 검증된 지식] |
| `--transition-speed` | 애니메이션 기본 속도 | `0.8s ease-out` | 부드럽고 전문적인 사용자 경험 제공. | [추측] |

#### 3. 섹션별 인터랙티브 컴포넌트 기술 사양 (Section Tech Specs)

**[A. 히어로 섹션: Pain Point Trigger & Hooking Title Card]**
*   **목표:** 사용자의 주의를 즉시 끌어내고, 기존 지식에 대한 문제 제기(Pain Point)를 통해 공감대를 형성한다.
*   **레이아웃 (90vw / 10vw 분할):**
    *   **좌측 영역 (Hooking Module - 85% 폭):** '시간 낭비', '불확실성' 등 부정적 감정을 자극하는 키워드를 반복 노출.
        *   *인터랙션:* 페이지 로드 시, 키워드가 무작위 간격(0.1s~0.3s)으로 **페이드 인 (Fade-In)** 되면서 '🚨 경고' 사운드 효과를 연동한다. [근거: Designer 개인 메모리]
        *   *Tech Spec:* `animation: keyphrase fadein 0.1s steps(1, end);`
    *   **우측 영역 (Solution Tease - 15% 폭):** "OEA 프로세스가 답입니다."라는 문구와 함께 작은 로딩 스피너를 배치하여, 해결책이 '시스템'에 의해 제공됨을 암시한다.
        *   *인터랙션:* 좌측의 키워드 노출이 최고조에 달할 때 (3초 경과), 우측 스피너가 **Scale Up/Down 애니메이션**을 거쳐 "Loading Solution..." 문구로 전환되며 다음 섹션으로 자연스럽게 유도한다.

**[B. OEA 프로세스 설명 섹션: The Core System Flow]**
*   **목표:** 우리가 제공하는 핵심 방법론('관찰→추출→자동화')을 시각적이고 시스템적으로 보여준다.
*   **레이아웃:** 가로 스크롤 방식 (Horizontal Scrolling) 또는 단계별 좌우 플로우(Left-to-Right Flow). [근거: Designer 개인 메모리]
*   **컴포넌트 기술 사양 (3단계):** 각 단계를 독립적인 인터랙티브 컴포넌트로 정의합니다.

| 요소 | 내용 | 애니메이션/상호작용 로직 (JS Trigger) | CSS 타이밍 가이드 |
| :--- | :--- | :--- | :--- |
| **[1] Observe Module** | 문제 상황 포착 및 데이터 영역 설정 (Pain Point 강조). | 사용자가 이 섹션에 진입하면, 배경의 노이즈 패턴(`radial-gradient`)이 미세하게 움직이기 시작함. 마우스 오버 시, '관찰' 박스의 테두리가 **Pulse 애니메이션**을 하며 다음 단계로 진행할 준비가 되었음을 알림. | `transition: border 0.3s ease-out;` / `@keyframes pulse { ... }` |
| **[2] Extract Module** | 핵심 데이터 추출 및 필터링 프로세스 설명 (기술적 과정). | '추출' 단계에 도달하면, 페이지 중앙의 가상 데이터 흐름도(Flowchart)가 좌측에서 우측으로 **데이터 로딩 애니메이션**을 보여주며 정보를 채움. 코다리 개발 결과물(`pack_apply.py`가 처리하는 로직)를 시각적으로 재현함. | `animation: data-load 1s cubic-bezier(0.25, 0.46, 0.45, 0.94);` |
| **[3] Automate Module** | 자동화 시스템의 최종 결과물 및 가치 제시 (Solution). | '자동화' 단계에 도달하면, 모든 애니메이션이 최고조에 달하며, 배경색이 일시적으로 가장 밝은 크림색(`var(--color-secondary-bg)`)으로 바뀌며 **'Aha! Moment'**를 연출. 여기에 최종 CTA 배너가 팝업처럼 나타남. | `transition: background-color 0.5s ease-in;` |

**[C. CTA/결제 유도 섹션: The Conversion Funnel]**
*   **목표:** 망설이는 사용자에게 마지막 신뢰감을 주고 즉각적인 구매를 유도한다.
*   **핵심 요소 (The Trust Bar):** "지금 결제하면 얻을 수 있는 이점 3가지" 목록화.
    *   *인터랙션:* 각 장점을 클릭할 때마다, 배경의 필름 노이즈가 잠시 사라지면서(Noise Fade Out), 마치 '진실'이 드러나는 듯한 효과를 주고 해당 내용에 시선이 집중되도록 유도한다.
    *   *기술 구현:* 이 섹션은 **API 연동을 전제로 한 카운터**를 배치하여, "현재 O명까지 등록 완료"와 같은 실시간 데이터를 보여줘야 합니다. [근거: Designer 개인 메모리]
