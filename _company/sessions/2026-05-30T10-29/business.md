## 2️⃣ KPI 지표 상세 정의 (KPI Definition)

### 📉 1. API 과금 비용 절감률 (Cost Reduction Rate)
시스템 안정화를 통해 불필요한 API 호출을 차단하거나 최적화하여 비용을 얼마나 줄였는지 측정합니다.

| 항목 | 세부 지시사항 (Formula) | 목표치 (Benchmark) | 측정 주기 |
| :--- | :--- | :--- | :--- |
| **비용 절감률** | `(기존_예상_비용 - 실제_발생_비용) / 기존_예상_비용 × 100` | **+20% 이상 절감** | 일별/주별 |
| **불필요 호출 차단율** | `총_호출_횟수 - (중복/실패_재시도_차단_횟수) / 총_호출_횟수 × 100` | **30% 이상 감소** | 실시간 로깅 기준 |
| **단가 최적화 비율** | `기존_API_가격 - 현재_효과적_단가 / 기존_API_가격 × 100` | **5% 이상 협상/최적화** | 월간 |

*   **비고:** 단순히 API 호출 횟수를 줄이는 것뿐만 아니라, 재시도 로직을 최적화하여 '실패 후 재호출'로 인한 중복 비용을 차단하는 것이 핵심입니다.
    `[근거: 현빈 개인 메모리 — 12 초~15 초 최적 타이밍 / 과금 최소화]`

### 🛡️ 2. 프로그램 안정성 지표 (System Stability Index)
시스템이 정상적으로 작동하여 비즈니스 중단 없이 서비스를 제공하는 능력을 수치화합니다.

| 항목 | 세부 지시사항 (Formula) | 목표치 (Benchmark) | 측정 주기 |
| :--- | :--- | :--- | :--- |
| **오류 발생률 (Error Rate)** | `에러_발생_횟수 / 총_작업_회수 × 100` | **0% 유지** (Critical 에러) | 실시간 로깅 기준 |
| **평균 처리 시간 (Avg. Latency)** | `총_처리_시간 / 총_작업_회수` | **기존 대비 -20%** | 일별 평균 |
| **자동 복구 성공률** | `성공적으로 복원된_에러_횟수 / 총_에러_횟수 × 100` | **95% 이상 복구** | 실시간 로깅 기준 |

*   **비고:** '오류 발생률'은 절대 0 에 가까워야 하며, 특히 파일 시스템 경로 오류 (예: `4858-i8id`) 가 재발하지 않는지 모니터링해야 합니다.
    `[근거: 지난 의사결정 로그 — 파일 시스템 접근 권한 및 경로 오류 수정]`
