# 📐 [Final Tech Spec] 관찰 → 추출 → 자동화 (OEA) 인터랙티브 컴포넌트 V2.0

## 🎯 목표 및 전제
*   **목표:** 단순한 정보 전달이 아닌, 사용자의 능동적 참여를 유도하는 '경험'을 제공한다. (직관적인 피드백 루프 필수).
*   **전제:** 이 컴포넌트는 `Intersection Observer`와 `Scroll-based Animation`에 의존하며, 모든 상호작용은 트래킹 이벤트(`trackEvent`)를 발생시켜야 한다.

## ✨ 단계별 인터랙티브 상세 명세서 (JS/CSS Logic)

### 1. 관찰 (Observe) - `O-Card`
*   **트리거:** 스크롤이 O-Card에 진입했을 때 (`Intersection Observer: visible`).
*   **기본 상태 (Default):** 뉴트럴 어스톤 배경, 핵심 질문("무엇을 놓치고 계신가요?")만 표시.
*   **상호작용 1: `:hover` (마우스 오버)**
    *   **CSS:** `box-shadow`의 확산 및 색상 변화 (`#B04D3A`로 변이). `transform: translateY(-5px);`.
    *   **JS/Event:** 없음. 순수 CSS 애니메이션으로 처리.
*   **상호작용 2: `onClick` (클릭)**
    *   **CSS:** 카드가 활성화된 상태(Active State)로 전환되며, 주변의 연결선 색상이 `#B04D3A`로 변함.
    *   **JS/Event:**
        1.  `trackEvent('O_Observe_Click')` 호출.
        2.  Output Display Area에 **'데이터 로딩 스피너(Loading Spinner)'** 컴포넌트를 띄우고, 배경색을 `#F5F3E9` (밝은 연한 베이지)로 변경한다.
        3.  이 상태는 다음 단계(`E-Card`)가 활성화될 때까지 유지되어야 함.

### 2. 추출 (Extract) - `E-Card`
*   **트리거:** O-Card 클릭 후, 스크롤을 E-Card로 이동시켰을 때.
*   **기본 상태:** 핵심 키워드("핵심 정보를 뽑아내는 방법").
*   **상호작용 1: `:hover` (마우스 오버)**
    *   **CSS:** O-Card와 동일하게 그림자/색상 변화.
    *   **JS/Event:** 없음.
*   **상호작용 2: `onClick` (클릭)**
    *   **CSS:** 활성화 상태로 전환, 주변 연결선 색상 변경.
    *   **JS/Event:**
        1.  `trackEvent('E_Extract_Click')` 호출.
        2.  Output Display Area의 로딩 스피너를 **'데이터 추출 필터링 UI (가상의 폼)'**로 대체한다. 이 UI는 사용자가 직접 '핵심 단어'를 입력할 수 있는 `<input>` 필드와 "분석 실행" 버튼을 포함해야 한다.
        3.  이 상태가 다음 단계(`A-Card`) 활성화 전까지 유지되어야 함.

### 3. 자동화 (Automate) - `A-Card`
*   **트리거:** E-Card에서 '분석 실행' 버튼 클릭 시, 스크롤을 A-Card로 이동시키거나(혹은 직접 다음 단계 로딩), 페이지의 마지막 섹션에 도달했을 때.
*   **기본 상태:** 결과 제시("자동화된 시스템 구축").
*   **상호작용 1: `onClick` (클릭)**
    *   **CSS:** 최종 활성화 상태로 전환, 주변 연결선 색상 변경.
    *   **JS/Event:**
        1.  `trackEvent('A_Automate_Click')` 호출.
        2.  Output Display Area의 '분석 실행' 버튼 클릭을 감지하고, **가상의 분석 결과 데이터(JSON 구조)**를 받아와 이를 시각적인 그래프 또는 요약 리스트 형태로 렌더링한다. (실제 백엔드 API 연결 지점 명시).
        3.  이 결과를 통해 최종 CTA 섹션으로의 전환이 이루어져야 함.
