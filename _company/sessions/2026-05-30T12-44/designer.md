# 🎨 Jinny`s Lab 비주얼 시스템 SOP V2.0: Chaos to Order

## 💡 1. 프로젝트 목표 및 컨셉 정의 (The Core Concept)
**목표:** 시청자가 현재 느끼는 '비효율적인 노력(Chaos)'에 대한 고통을 극대화하여, 우리 회사가 제시하는 '체계적이고 자동화된 시스템(Order)'으로의 전환 필요성을 감성적으로 자극한다.

**테마 비주얼 해석:**
*   **CHAOS (혼돈):** 예측 불가능한 요소, 과부하, 오류, 수작업의 피로감.
    *   *비주얼:* 무질서한 텍스트 블록(오버레이), 빨간 경고 아이콘/라인, 빈티지 필름 노이즈 증폭, 엉킨 선형 구조도.
    *   *색상:* **Alert Red** (`#D9534F` - Warning Tone) 또는 오염된 채도가 높은 주황 계열.
*   **ORDER (질서):** 명확한 흐름, 시스템의 안정성, 자동화의 간결함.
    *   *비주얼:* 깔끔한 플로우차트, 좌우 대칭 구조, 데이터가 유기적으로 연결되는 빛나는 라인/노드.
    *   *색상:* **Tech Blue** (`#4A90E2` - Trust Tone) 또는 세련된 네온 계열의 청록색.

### 2.2. 타이포그래피 및 레이아웃 규칙
*   **Headline / Hook:** *Brush Script* (또는 유사 캘리그래피체). **[규칙]:** 반드시 Chaos/Order 전환 시점에만 사용하며, 문장의 감성적 클라이맥스를 담당.
*   **Body / System:** *Pretendard*. **[규칙]:** 시스템 설명 및 프로세스 단계(OBSERVE/EXTRACT/AUTOMATE)를 명료하게 전달할 때 일관되게 사용.
*   **Callout / Detail:** *Courier New* (또는 유사 모노 스페이스체). **[규칙]:** 코딩, 데이터 레이블, API명 등 '기술적' 정보를 제공하는 곳에만 제한적으로 사용하여 전문성을 극대화.

### 2.3. 핵심 인터랙션 컴포넌트 정의 (Tech Spec Focus)
모든 요소는 정지된 이미지가 아닌 **상태 변화(State Change)**를 중심으로 설계되어야 합니다.

1.  **[Component: Pain Point Trigger Card]**
    *   **목적:** 영상 초반 3초, 시청자의 공감대를 형성하며 '문제 제기'하는 화면.
    *   **레이아웃:** 전체 배경에 `radial-gradient`를 이용한 미세 노이즈 레이어 (`opacity: 0.1`) 오버레이 유지.
    *   **핵심 애니메이션 (Chaos State):** 화면 중앙에서 여러 개의 **랜덤하게 배치된 텍스트 블록과 빨간 경고 아이콘(`Alert Red`)**이 무작위로 '점프-겹치기' 효과를 일으키며 진입 (`ease-in-out`, `duration: 0.2s`).
    *   **트리거:** 영상 시작 시 강제적으로 발동 (스크롤/마우스 오버 반응 불필요).

2.  **[Component: Process Flow Diagram]**
    *   **목적:** OBSERVE → EXTRACT → AUTOMATE의 흐름을 시각화하는 메인 인포그래픽.
    *   **레이아웃:** 좌측(Chaos)과 우측(Order)으로 구조 분할 (`flex` 또는 `grid` 사용 권장).
    *   **핵심 애니메이션 (State Change):**
        1.  **OBSERVE 진입:** 배경에서 빨간 경고색의 복잡한 선이 무질서하게 얽히며 나타남 (Chaos State, 노이즈).
        2.  **(트리거: '자동화' 키워드 등장 시)** 모든 요소가 순간적으로 정지하고, **Tech Blue** 색상의 깨끗한 플로우 라인이 중앙을 가로지르며 기존의 얽힌 선들을 깔끔하게 분리/재정렬함 (Order State).
        3.  **데이터 흐름:** 각 단계별 데이터 블록은 `opacity`를 이용해 순차적으로 페이드 인하며, 연결되는 화살표는 `stroke-dasharray` 애니메이션을 통해 부드럽게 그려지도록 구현.
