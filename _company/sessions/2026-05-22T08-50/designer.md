# ⚠️ System Error State Design Kit & Tech Spec Fragment (V1.0)

## 🎯 목표 및 원칙
이 컴포넌트의 최우선 목표는 **'사용자의 신뢰 유지'**입니다. 단순한 '에러 코드'를 보여주는 것이 아니라, 시스템 오류로 인한 *일시적 불편함*을 인지시키고, 재시도(Retry)를 유도하는 명확하고 따뜻한 사용자 경험(UX)을 제공해야 합니다.

### 1. 비주얼 가이드라인 (Visual Spec)
| 요소 | 사양/값 | CSS 변수 | 디자인 의도 및 근거 |
| :--- | :--- | :--- | :--- |
| **전체 배경** | `var(--color-bg-secondary)` (`#F5EFEA`) | `--color-error-fallback` | 기존의 종이 질감과 유사하게 유지하여 이질감을 최소화합니다. (단, 에러가 발생했음을 알리기 위해 미세한 노이즈 오버레이를 강화) [근거: Designer 검증된 지식] |

| **CTA 버튼** | `Secondary BG`와 유사하지만, 클릭 유도가 명확한 미디엄 톤의 배경색 (`#A08C7E`). | `--color-button-retry` | "다시 시도하기" 버튼은 가장 눈에 잘 띄면서도 브랜드의 따뜻한 톤을 유지해야 합니다. [근거: Designer 개인 메모리] |
