### 📺 썸네일 비주얼 시스템 확정 및 Tech Spec

우리가 만드는 것은 단순한 '이미지'가 아니라, 시청자의 호기심을 자극하고 클릭을 강제하는 **‘상호작용하는 프롤로그(Interactive Prologue)’**입니다. 따라서 모든 요소는 정적인 PNG 형태가 아닌, 다음과 같은 구조를 가진 인터랙티브 프로토타입 형태로 정의합니다.

#### 1. [썸네일 기본 프레임 시스템]
*   **규격:** 모바일 최적화 (9:16 비율)
*   **전체 배경 레이어:** 뉴트럴 어스톤 (`#DDC2B9`) + 빈티지 필름 노이즈 오버레이. [근거: Designer 검증된 지식]
*   **구조 원칙:** 3분할 구조 (좌/중앙/우)를 유지하며, 각 영역은 정보의 **'상태(State)'**를 나타냅니다.

| 영역 | 역할 (콘텐츠 기능) | 시각적 정의 및 애니메이션 목표 | Tech Spec 지침 |
| :--- | :--- | :--- | :--- |
| **좌측 (Pain Point)** | 문제 제기/후킹 문구 배치. 시청자의 '불안감' 자극. | 배경에 흐르는 미세한 데이터 스트림(녹색 네온). 핵심 질문은 텍스트가 아닌, 오류 메시지 형태 (`[ERROR: Input Data Missing]`)로 노출되어야 함. | **애니메이션:** `text-shadow`와 함께 타이핑 효과(`typewriter effect`). 데이터 라인은 지속적인 미세한 움직임 (`@keyframes scanline`). |
| **중앙 (Core Concept)** | 핵심 주장/해결책 제시. 채널의 지식 전달 영역. | 시스템 구조도(OEA) 플로우차트를 간소화하여 배치. 프로세스 단계는 '시스템 활성화' 상태로 강조. | **애니메이션:** 각 노드가 순차적으로 점멸하며 나타남 (Fade-in + Glow). 핵심 단어만 네온 컬러로 하이라이트 (`var(--color-primary-neon)`). |
| **우측 (CTA/Callout)** | 해결책의 가치 증명 및 행동 유도. | '솔루션 제시'라는 명확한 메시지(예: "시스템을 재설정하세요")를 강조하며, 구매 전환 유도를 위한 작은 데이터 패널 배치. | **애니메이션:** 마치 시스템이 정상 작동하는 듯한 안정적인 네온 그리드 패턴. CTA 버튼은 마우스 오버 시 강렬하게 깜빡이는 효과(`pulse` animation). |

#### 2. [핵심 시각 효과: 글리치 노이즈 (Glitch Noise) Tech Spec]
글리치는 단순히 '노이즈'가 아니라, **'시스템의 일시적 오류 상태'를 보여주는 시각적 증거**로 활용되어야 합니다.

*   **발생 조건:** 콘텐츠의 핵심 메시지가 전환되거나, 문제점을 지적할 때 (특히 좌측 영역) 사용합니다. [근거: 지난 의사결정 로그]
*   **CSS/JS 구현 명세:**
    1.  **시점 정의:** 특정 키프레임(예: 0초 $\rightarrow$ 0.3초 구간).
    2.  **트리거:** JavaScript를 통해 `isGlitching` 플래그가 참이 될 때 작동합니다.
    3.  **효과 구현 (CSS):**
        *   **Chromatic Aberration:** 요소의 색상 채널(R, G, B)을 미세하게 분리하여 겹치게 합니다 (`filter: drop-shadow(...)`).
        *   **Horizontal Shift:** `clip-path`를 사용하여 화면을 수평으로 여러 번 잘라내고(`slice`), 각 조각에 아주 짧은 시간차의 강제 이동(e.g., `-2px`, `+2px`) 애니메이션을 적용합니다.
        *   **Timing:** 전체 효과는 0.1초 간격의 연속적인 깜빡임으로 구성되며, 반복 속도는 빠르게 설정해야 합니다 (`animation: glitch 0.1s infinite alternate`).
