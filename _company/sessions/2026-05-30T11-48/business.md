# 📊 API 과금 비용 절감률 측정 KPI 달성 로드맵 (v1.0)
**작성자:** 💼 현빈 (Head of Business) | **작성일:** 2026.05.30 | **상태:** 설계안 기반 구체화 완료

## 1. 개요 및 목적
Jinny`s Lab 의 자동화 시스템 (`4858-i8id`) 은 API 호출 시 과금이 발생합니다. 개발팀 (코다리) 이 안정화를 통해 파일 접근 오류를 수정하고 로깅 프로세스를 도입한 시점부터, **API 사용 효율성**과 **불필요한 과금 방지** 효과를 정량적으로 측정해야 합니다.

본 로드맵은 코다리팀의 실시간 로깅 데이터 수집을 전제로, API 과금 비용을 절감하는 구체적인 KPI 와 달성 기준점을 정의합니다.

## 2. 핵심 원칙 (CEO 지시 반영)
> **"비개발자의 자동화 프로그램"이므로, 복잡한 로그 분석보다는 '결과 지표'에 집중해야 합니다.**
- **투명성:** 비용 발생 내역을 누구나 읽을 수 있어야 함.
- **최소화:** 불필요한 API 호출은 즉시 차단되고 그 효과가 기록되어야 함.

## 3. KPI 정의 및 측정 로직 (코다리팀 실행 요청 사항)

### 🎯 Primary KPI: 비용 절감률 (Cost Saving Rate)
불필요한 호출을 막아 얼마나 비용을 아꼈는지를 나타냅니다.
- **공식:** `(기대 과금액 - 실제 과금액) / 기대 과금액 × 100%`
- **측정 주기:** 실시간 (Daily Log Aggregation)
- **성공 기준:** 30 일 평균 절감률 **> 20%** 달성

### 🎯 Secondary KPI: 무효 호출 차단율 (Invalid Call Block Rate)
오류 발생이나 중복 요청으로 인한 과금을 얼마나 막았는지 나타냅니다.
- **공식:** `무효 호출 횟수 / 총 API 호출 횟수 × 100%`
- **성공 기준:** 파일 접근 오류 발생 시 **100%** 차단 (Zero Tolerance)

### 🎯 Tertiary KPI: 과금 리스크 경고 시간 (Alert Latency)
과금이 예상되는 시점보다 얼마나 빨리 감지했는지 나타냅니다.
- **성공 기준:** 60 초 이내 경고 발송
