## 💡 개요 및 목표
본 라이브러리는 [관찰 → 추출 → 자동화 (OEA)] 프로세스 기반의 모든 콘텐츠(유튜브 쇼츠, 인트로 카드, 썸네일)에 통일성을 부여하고, 제작된 에셋이 **'상태 변화(State Change)'**를 통해 시청자의 몰입도를 극대화하는 것을 목표로 합니다.

**[핵심 원칙]**
*   **재질:** 뉴트럴 어스톤 (Primary BG: `#DDC2B9`) + 빈티지 필름 노이즈 오버레이 (`opacity: 0.1`). [근거: Designer 검증된 지식]
*   **전달 방식:** 정적 이미지 X → **인터랙티브 시퀀스/모바일 목업 비디오** O. [근거: Designer 개인 메모리 / Self-RAG]
*   **시스템 기반:** 모든 요소는 CSS 변수 (`var(--color-...)`, `var(--size-...)`)로 정의되어야 합니다.

### 1. 핵심 컴포넌트 3종 (모바일 최적화, 9:16 비율)

| 에셋 명 | 용도 | 기술 구현 목표 (Tech Spec) |
| :--- | :--- | :--- |
| **① 후킹 타이틀 카드** | 영상 초반 3초 이내 시청자 주의 집중 유도. | 강렬한 캐리커처/질문 제시. 배경은 어스톤, 전경에 손글씨체(`Brush Script` 등)로 질문을 과감하게 배치하고, **'왜?'라는 의문을 던지는 타이핑 효과(Typewriter Effect)** 애니메이션 적용. [근거: Designer 검증된 지식 / Writer] |
| **② OEA 프로세스 인포그래픽** | 콘텐츠의 핵심 가치 (Observe $\rightarrow$ Extract $\rightarrow$ Automate) 설명 시퀀스. | 정적인 흐름도가 아닌, **데이터 로딩 애니메이션(Loading Animation)**이 필수. '관찰' 단계 진입 시 배경색이 미세하게 어두워지고, 데이터 추출 과정에서 텍스트가 한 글자씩 순차적으로 나타나며 신뢰도를 높임. [근거: Designer 개인 메모리 / Self-RAG] |
| **③ 최종 CTA/결제 유도 배너** | 콘텐츠 말미의 행동 유도 (클릭/구매). | **'Pain Point 자각 → Solution 제시 → 행동 촉구' 3단계 플로우.** 버튼은 단순한 사각형이 아닌, 누르는 듯한 미세한 물리적 피드백(Press Down Effect) 애니메이션을 적용하여 클릭률 극대화. [근거: Self-RAG (CTA V3.0)] |

### 2. 상세 기술 명세서 (Tech Spec) 업데이트 및 통합

#### A. 후킹 타이틀 카드 구현 가이드라인
*   **사이즈:** 모바일 최적화 (1080px x 1920px).
*   **폰트 계층 구조:**
    *   Headline: `Brush Script` (제목/질문) - 색상: `#3A3530`
    *   Sub-text: `Pretendard` (보조 설명) - 색상: `#3A3530`
    *   Source Indicator: `Courier New` (출처/근거) - 색상: `#6b645d`
*   **애니메이션:** 텍스트가 화면에 진입할 때, **'타이핑 효과(Typewriter Effect)'**를 필수적으로 적용해야 합니다. 타이밍은 0.3초 간격으로 설정합니다.

#### B. OEA 플로우차트 인터랙티브 명세 (핵심)
*   **좌측 모듈 (Pain Point Trigger):** 사용자의 문제점을 상기시키는 역할을 하며, 마우스 오버 시 미세한 진동 애니메이션(Vibration)을 주어 주의를 끕니다. [근거: Designer 개인 메모리]
*   **중앙 플로우 (OEA):** 각 단계(`Observe`, `Extract`, `Automate`)는 클릭 가능한 컴포넌트가 되어야 합니다. **클릭 시, 다음 섹션으로 부드럽게 스크롤 이동(Smooth Scroll)**하며 해당 프로세스의 원리 설명이 팝업됩니다.
*   **기술적 요구사항:** 모든 플로우 박스는 `:hover`와 `:active` 상태를 정의해야 하며, 이 전환 과정에 `transition-duration: 0.5s;` 가 적용되어야 합니다.
