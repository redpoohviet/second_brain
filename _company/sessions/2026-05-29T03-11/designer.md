#### 📄 최종 Tech Spec: OEA 프로세스 흐름 컴포넌트 (V2.0)
**(적용 대상: 쇼츠 영상의 핵심 정보 전달 구간)**

**1. 기술 목표:** 사용자의 시선이 '문제 제기(Pain Point)' $\rightarrow$ '해결책 제시(OEA Process)' 순으로 자연스럽게 이동하며, 과정 설명에 몰입하게 만드는 **상태 기반 애니메이션 경험**을 제공합니다.
**2. 구조 (좌우 분할 유지):**
*   **L-Panel (Pain Trigger):** 초기 시선을 사로잡는 '문제'를 간결하게 제시하는 모듈.
*   **R-Panel (Solution/OEA Flow):** 3단계(Observe, Extract, Automate)의 과정 설명이 인터랙티브하게 전개되는 메인 영역.

**3. 컴포넌트 상세 정의:**

| 요소 | 속성 (CSS Variable / Value) | 상호작용 로직 (JS Trigger Point) | 애니메이션 가이드 (Timing/Easing) |
| :--- | :--- | :--- | :--- |
| **전체 배경** | `var(--color-bg-secondary)` + 미세 노이즈 (`opacity: 0.1`) | 페이지 로드 시 부드러운 페이드인. | `transition-duration: 1s;` / `ease-out`. |
| **[A] Pain Point (L-Panel)** | `--color-text-dark`, 2~3줄의 짧은 질문/통계 자료. | 페이지 로드 후 $\mathbf{0.5s}$ 지점에서 강하게 등장(Staggered Fade-in). | `transform: translateY(-10px);` $\rightarrow$ `opacity: 1;`. (빠르고 충격적인 느낌) |
| **[B] OEA Step 1: Observe** | `--color-text-dark`, 아이콘, 간단한 설명. | A가 등장한 후 $\mathbf{1.0s}$ 지점에서 시작. 클릭 또는 다음 섹션 전환 시 발동. | **(핵심):** 좌측에서 우측으로 슬라이드 인(`transform: translateX(-100%)` $\rightarrow$ `translateX(0)`). 배경에 미세한 '관찰망' 애니메이션 오버레이 (`background-size: 200%`). |
| **[C] OEA Step 2: Extract** | `--color-text-dark`, 과정 설명, 예시 코드 블록. | B가 완료된 후 $\mathbf{1.5s}$ 지점에서 시작. (시간적 간격 확보). | **(핵심):** 수직적으로 단계가 쌓이는 효과 (`opacity: 0` $\rightarrow$ `opacity: 1`). 코딩 서체(`Courier New`)는 '타이핑' 애니메이션 효과 적용 필요. |
| **[D] OEA Step 3: Automate (CTA 연결점)** | `--color-text-dark`, 최종 결과물 예시, 핵심 가치 강조. | C가 완료된 후 $\mathbf{2.0s}$ 지점에서 시작. 가장 느리고 확신에 찬 등장. | **(핵심):** 전체 섹션이 오른쪽에서 부드럽게 커지며 (`scale: 0.9` $\rightarrow$ `scale: 1.0`) 최종 메시지를 강조. 다음 단계(CTA)로의 연결을 시사하는 '활성 영역' 강조 (밝은 배경색 깜빡임 효과). |

**4. 코다리님께 요청하는 API/JS Hook 포인트:**
1.  `trigger_section_A()`: Pain Point가 등장할 때 호출되어야 합니다.
2.  `trigger_step_B(elementId)`: 'Observe' 단계를 시각적으로 활성화하고, 관련 텍스트를 불러와 애니메이션을 시작해야 합니다.
3.  `progress_to_next_section()`: OEA 프로세스 전체가 끝났음을 인지시키고, 최종 CTA 섹션으로 부드럽게 전환(Transition)하는 함수입니다.
