## II. 🎬 콘텐츠별 핵심 에셋 기술 사양서 (Tech Spec)

### A. 후킹 타이틀 카드 (Hooking Title Card)
*   **목적:** 시청자의 'Pain Point'를 강하게 자극하여 이탈을 막고, 다음 정보에 대한 기대를 유발한다.
*   **포맷:** 모바일 최적화 9:16 비율 (예시 크기: 1080px x 1920px).
*   **애니메이션 로직 ( 필수 ):**
    1.  **진입 (In):** 배경은 미세한 빈티지 필름 노이즈가 점차 페이드 인하며 시작 (`opacity: 0` $\rightarrow$ `opacity: 1`, duration: 0.5s, easing: ease-in-out).
    2.  **Hook Text 등장:** 메인 질문/슬로건(`Brush Script`)은 화면 중앙 상단에서 **바운스 효과(Bounce In)**와 함께 나타난다 (Timing: 0.3s). [근거: Designer 개인 메모리]
    3.  **Pain Point 제시:** 구체적인 문제점 목록(Bullet Points)은 좌측에서 **좌->우로 슬라이드 인**되며 순차적으로 등장한다 (Staggered animation, duration: 0.4s per point).
*   **개발 요구사항:** `var(--color-text-dark)`를 사용하여 배경의 어스톤 색상 위에서도 가독성을 최대화해야 한다.

### B. OEA 프로세스 인포그래픽 (Observe $\rightarrow$ Extract $\rightarrow$ Automate)
*   **목적:** 채널의 핵심 방법론을 시각적으로 체계화하고, '지식'이라는 추상적인 개념에 물리적 구조를 부여한다.
*   **포맷:** 9:16 비율 / 단계별 플로우차트 (Flowchart).
*   **핵심 컴포넌트:** 세 개의 상호 연결된 박스(`Observe`, `Extract`, `Automate`). [근거: Designer 개인 메모리]
*   **상호작용 로직 (Critical):**
    1.  **초기 상태:** 세 단계는 좌우로 배치되며, 초기에는 투명도가 낮은 회색 박스로 존재한다.
    2.  **액션 유도 (Interaction):** 시청자가 **'Observe'** 영역을 볼 때(혹은 해당 키워드에 도달했을 때), 이 박스는 `var(--color-bg-secondary)`로 밝게 변하고, 미세한 빛나는 테두리 효과(`box-shadow`)가 발생한다.
    3.  **데이터 로딩 시뮬레이션:** 'Extract' 단계에서 데이터가 추출되는 과정은 단순 텍스트 노출이 아닌, **좌측에서 우측으로 데이터를 전송하는 애니메이션 파이프라인(Pipe Animation)**을 통해 보여주어야 한다. [근거: Designer 개인 메모리]
*   **개발 요구사항:** 각 컴포넌트의 경계선과 내부 요소는 반드시 CSS 변수를 사용해야 하며, 데이터 흐름은 JS 이벤트 리스너를 통한 상태 변화로만 구현되어야 한다.

### C. CTA 및 결론 에셋 (Call to Action)
*   **목적:** 시청자가 행동(구독/구매/학습)하도록 유도하는 클라이맥스 장면.
*   **포맷:** 9:16 비율.
*   **애니메이션 로직:**
    1.  **강조:** 화면 전체에 걸쳐 **'Opportunity Cost (기회비용)'**라는 키워드가 반복적으로 플래시(Flash)되는 효과를 사용한다. (높은 긴장감 유도).
    2.  **명확한 행동 지침:** CTA 버튼 영역은 단순 텍스트가 아닌, 마치 '시스템의 출력 결과물'처럼 **정교하게 빛나고 진동하는 UI 요소**로 디자인되어야 한다. [근거: Designer 개인 메모리]

## III. 🚀 에셋 구현을 위한 통합 기술 가이드라인 (For Developer/Codari)
1.  **파일 구조:** 모든 컴포넌트(Title Card, OEA Flow 등)는 별도의 React Component 또는 Web Component로 분리되어야 한다. (재사용성 극대화).
2.  **애니메이션 라이브러리:** GSAP이나 Framer Motion 같은 전문 애니메이션 라이브러리를 사용하여 모든 키프레임과 타이밍을 관리한다. `transition-duration: 0.5s;` 및 `ease-out`은 기본값으로 적용된다.
3.  **최적화 원칙 (Performance):** 모든 인터랙션 요소는 배경 이미지 사용을 지양하고, SVG 또는 CSS 애니메이션만을 사용하여 로딩 속도와 시스템 안정성을 최우선으로 한다.
