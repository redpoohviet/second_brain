# 🎨 Jinny`s Lab 비주얼 시스템 가이드북 (Master Visual System Guidebook V1.0)

> **목표:** 모든 숏폼 콘텐츠(유튜브 쇼츠, 인스타그램 릴스, 캐러셀)에 일관성 있고 전문적인 '시스템적 신뢰감'을 부여하는 것을 목표로 합니다.
> **핵심 원칙:** 본 가이드의 모든 에셋은 **정지된 그림(Static Image)**이 아닌, 사용자의 액션과 시간 경과에 따라 변화하는 **상호작용 시스템 컴포넌트 (Interactive System Component)**를 기본으로 설계합니다.

## 🧩 1. 디자인 토큰 (Design Tokens) 정의
모든 값은 CSS 변수(CSS Variables) 형태로 관리하여, 브랜드 수정 시 전역 변경이 용이하도록 합니다.

### A. 컬러 팔레트 (Color Palette)
| 역할 | 명칭 | HEX 코드 | CSS 변수 | 사용 목적 및 감성 | 근거 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Primary BG** | 뉴트럴 어스톤 | `#DDC2B9` | `--color-bg-primary` | 채널 배너 전체 배경. 안정감, 아날로그적 신뢰감을 유지하며 주 색상으로 활용. | [근거: Designer 검증된 지식] |
| **Secondary BG** | 밝은 크림색 | `#F5EFEA` | `--color-bg-secondary` | 정보 블록이나 템플릿 내부의 구분 영역. 대비를 통해 가독성 확보 및 '종이 질감' 느낌 부여. | [근거: Designer 검증된 지식] |
| **Text Dark** | 짙은 차콜 그레이 | `#3A3530` | `--color-text-dark` | 모든 본문, 제목의 기본 텍스트 색상. 블랙보다 부드러워 가독성 높임. | [근거: Designer 검증된 지식] |
| **Accent Color** | (선택적) 시그널 포인트 | `#A35F42` | `--color-accent` | 강조, 경고(⚠️), CTA 버튼 등 사용자의 행동을 유도하는 부분에만 제한적으로 사용. | [추측] |
| **System Error** | 오류/위험 알림 | `#C04B4B` | `--color-error` | '시스템 오류 진단' 섹션에서 문제 발생 지점을 시각화할 때 사용 (빨간색 계열). | [근거: 지난 의사결정 로그] |

### B. 타이포그래피 가이드 (Typography System)
| 역할 | 용도 및 스타일 | 추천 폰트 (Primary/Fallback) | CSS 변수 | 적용 원칙 | 근거 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **H1 / Hook Title** | 메인 슬로건, 시선 강탈 문구. 감성적 연결 유도. | *Brush Script* (또는 유사 캘리그라피체) | `--font-hook` | 크고 가독성이 높아야 하며, Hand-drawn 느낌을 강조. | [근거: Designer 검증된 지식] |
| **Body / System Text** | OEA 프로세스 설명, 학습 내용, 자막의 핵심 정보 전달. (가장 많은 영역) | *Pretendard* 또는 *Noto Sans KR* | `--font-body` | 모던하고 기술적이며 가독성이 높은 산세리프체. 명확한 시스템 구조 확립이 목적. | [근거: Designer 검증된 지식] |
| **Code / Detail** | 데이터 레이블, 기술 용어 (OEA), 플로우차트의 작은 라벨링. | *Courier New* 또는 모노 스페이스 서체 | `--font-code` | 전문성 및 '시스템' 느낌 부여. 얇은 두께(Weight: Thin)를 추천. | [근거: Designer 검증된 지식] |

## 📐 2. 레이아웃 그리드 시스템 (Layout Grid System)
모든 콘텐츠는 모바일 최적화(Mobile-First) 원칙을 따릅니다.

### A. 기본 규격
| 유형 | 비율 | 크기 가이드라인 | 활용 예시 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **숏폼 영상** | 9:16 (Vertical) | 1080px x 1920px | 유튜브 쇼츠, 인스타그램 릴스 | 가장 주력하는 포맷. 자막 영역(Safe Zone) 확보 필수. |
| **피드 이미지/캐러셀** | 1:1 (Square) | 1080px x 1080px | 인스타그램 피드용 고정 이미지, 요약본. | 핵심 메시지만 추출하여 간결하게 구성. |

### B. 그리드 구조 정의
*   **좌측 영역 (Pain Point Trigger):** 사용자의 문제점을 상기시키는 시각적 장치 배치 공간. (전체 폭의 약 30%) [근거: Designer 개인 메모리]
*   **우측 영역 (Solution/Process Flow):** 해결책(OEA)과 학습 내용이 진행되는 핵심 정보 흐름도 배치 공간. (전체 폭의 약 70%) [근거: Designer 개인 메모리]

## ✨ 3. 필수 시각 컴포넌트 목록 및 기술 명세서 (Tech Spec)

자동화 시스템 구동을 위해 반드시 필요한 핵심 에셋 템플릿 3가지입니다. 이들은 단순한 배경이 아닌, **'정보를 전달하는 주체(Actor)'**로 설계되어야 합니다.

### Component 1: 후킹 타이틀 카드 (Hooking Title Card)
*   **목표:** 시청자의 스크롤을 멈추게 하고 영상에 대한 기대감을 최대로 끌어올리는 역할.
*   **레이아웃:** 중앙 집중형, 강렬한 대비 사용.
*   **기술 명세서:**
    *   **사이즈:** 9:16 (영상 시작 시 3~5초 노출)
    *   **배경:** `--color-bg-primary` 위에 미세한 빈티지 필름 노이즈 레이어 오버레이.
    *   **폰트:** 제목은 `--font-hook`, 부연 설명은 `--font-body`.
    *   **애니메이션 (필수):** 텍스트가 마치 **'타자기 타이핑 효과(Typewriter Effect)'**로 등장하거나, **'페이드 인과 동시에 확대되는(Scale & Fade In)'** 동작을 적용하여 정적인 느낌을 제거해야 함.

### Component 2: 프로세스 플로우차트 (OEA Flowchart)
*   **목표:** '관찰 → 추출 → 자동화'의 기술적 과정을 논리적으로 증명하는 핵심 인포그래픽.
*   **레이아웃:** 좌우 구조 분할(30%: 문제 제시 / 70%: 해결책 흐름).
*   **기술 명세서:**
    *   **사이즈:** 9:16 (영상 본론에서 가장 많은 시간 할애)
    *   **컴포넌트 정의:** 각 단계(Observe, Extract, Automate)는 별도의 카드 형태 컴포넌트로 분리되어야 함.
    *   **상호작용 로직 (필수):** 정보가 **순차적으로 노출(Sequential Reveal)**되도록 설계하며, 이전 단계가 완료될 때 다음 단계의 카드가 `0.5s ease-out` 애니메이션으로 슬라이드 인 되어야 합니다. (마치 시스템이 작동하는 것처럼).

### Component 3: CTA 및 가이드 안내 화면 (Call to Action Screen)
*   **목표:** 시청자가 명확하게 다음 행동(링크 클릭, 구독 등)을 취하도록 유도.
*   **레이아웃:** 미니멀리즘 강조, 단 하나의 메시지에 집중.
*   **기술 명세서:**
    *   **사이즈:** 9:16 (영상 종료 시점)
    *   **요소:** 핵심 문구(CTA), 행동 유도 버튼/영역, 로고/브랜드명.
    *   **애니메이션:** 모든 요소가 한 번에 등장하기보다, **'핵심 CTA 메시지 → 하단 링크 영역 → 로고'** 순서로 시간차를 두고 페이드 인 되어야 시각적 집중도가 높아집니다.
