### I. 🎨 브랜드 및 컴포넌트 기본 사양 (Global Variables)

모든 시각적 요소는 다음 CSS 변수를 기반으로 정의됩니다. [근거: Designer 검증된 지식]

| 영역 | 용도/기능 | 값(Hex Code) | CSS Variable | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| **Primary BG** | 주 배경 (안정감, 아날로그 감성) | `#DDC2B9` | `--color-bg-primary` | 전체 페이지와 썸네일의 기본 바탕. |
| **Text Dark** | 본문/제목 텍스트 | `#3A3530` | `--color-text-dark` | 블랙 대신 사용되는 차분한 메인 폰트 색상. |
| **Secondary BG** | 정보 블록 배경 (구분) | `#F5EFEA` | `--color-bg-secondary` | 강조된 섹션, 카드 내부 등 대비가 필요한 영역의 배경. |
| **Accent/Hook** | CTA 버튼, 핵심 하이라이트 | `#8B4513` (따뜻한 브라운) | `--color-accent-hook` | 사용자의 행동을 유도하는 가장 중요한 색상. |

### II. 💻 자동화 스크립트 구현 컴포넌트 상세 명세 (Component Specs)

자동화 스크립트는 다음 세 가지 핵심 인터랙티브 컴포넌트로 분할되어야 합니다. 이들은 정적인 이미지가 아닌, **상태 변화(State Change)**를 가진 컴포넌트여야 합니다. [근거: Designer 개인 메모리]

#### 1. 🎬 후킹 타이틀 카드 (Hooking Title Card)
*   **목적:** 시청자의 주의 집중 및 문제점(Pain Point) 즉시 제기.
*   **레이아웃:** 좌측에 Pain Point Trigger, 우측에 Solution/OEA 프로세스 간의 대비 구조. [근거: Designer 개인 메모리]
*   **애니메이션 로직 (Interaction):**
    1.  (State 0 - 초기) 배경은 뉴트럴 어스톤 위에 미세한 빈티지 필름 노이즈 오버레이 (`opacity: 0.1` radial-gradient). [근거: Self-RAG]
    2.  (Transition A - 등장) 타이틀(`--font-family-heading`)이 화면 상단에서 `ease-out` 효과로 **페이드 인 (Fade In)** 하며 나타남. (`transition-duration: 0.5s`).
    3.  (State 1 - 후킹) Pain Point 문구는 붉은색 계열의 애니메이션 강조 처리 후, 배경 전체에 노이즈 레이어가 순간적으로 증폭되었다가 감소하는 **시각적 자극**을 주어야 함. (JS 이벤트 리스너 필요).
    4.  (State 2 - 전환) "하지만 해결책이 있습니다."라는 문구와 함께 좌우 구조 분할이 발생하며 다음 컴포넌트로 자연스럽게 유도됨.

#### 2. ⚙️ 프로세스 설명 인포그래픽 (OEA Flowchart Component)
*   **목적:** '관찰→추출→자동화'의 단계를 시스템적으로, 이해하기 쉽게 전달.
*   **레이아웃:** 수평 또는 수직 플로우 차트 형태로, 각 단계(Step)가 카드가 되어 구성됨. [근거: Designer 개인 메모리]
*   **애니메이션 로직 (Interaction):**
    1.  스크롤 반응형 애니메이션 필수. 사용자가 스크롤할 때마다 다음 Step이 **좌측에서 슬라이드 인 (Slide-in)** 되며 나타나야 함. (`transition-duration: 0.6s`). [근거: Designer 개인 메모리]
    2.  **Step Card:** 배경은 `--color-bg-secondary`를 사용하고, 각 Step의 제목(Headline)과 설명(Body)을 명확히 분리함.
    3.  **상호작용 (Hover/Click):** 특정 Step 카드 위에 마우스 오버 시 (`:hover`), 해당 카드의 테두리가 `--color-accent-hook` 색으로 빛나고, 내부의 핵심 키워드(예: '데이터 추출')에 미세한 확대 애니메이션이 적용되어야 함. (JS 이벤트 리스너 필수).

#### 3. 🛒 최종 CTA 및 구매 유도 섹션 (Enrollment Component)
*   **목적:** 수익화 퍼널의 마무리를 담당하며, 결제를 최우선 행동으로 유도. [근거: Self-RAG]
*   **레이아웃:** 중앙 정렬을 기본으로 하며, 가격 단계별($27 / $49~$99 / $150+) 컴포넌트를 명확히 분리합니다.
*   **애니메이션 로직 (Interaction):**
    1.  **가격 비교 애니메이션:** 각 가격 패키지 박스(Card)는 순서대로 **카운트업(Count-up)** 효과로 금액이 나타나야 합니다. (예: $27, $49...가 0부터 카운팅되는 느낌).
    2.  **CTA 버튼 활성화:** '지금 구매하기' 버튼은 초기에는 `opacity: 0`으로 숨겨져 있다가, 모든 가격 정보 로딩이 완료된 후 JavaScript를 통해 점진적으로 `opacity: 1`로 나타나며 사용자의 행동을 유도합니다. (상태 변화 강제).
    3.  **결과물:** 이 섹션은 코다리가 개발할 **`specs/final_enrollment_techspec.md`** 파일에 정의된 로직을 시각적으로 구현하는 최종 지침서입니다. [근거: Self-RAG]
