## 📐 공통 컴포넌트 명세 (Mobile First: 9:16)

### 1. 후킹 타이틀 카드 (Hooking Title Card)
*   **목표:** 시청자의 주의를 즉각적으로 사로잡는 첫 화면 에셋.
*   **규격:** `width: 1080px`, `height: 1920px` (`var(--size-viewport)`).
*   **레이아웃 (Tech Spec):**
    *   배경: `radial-gradient(at top left, var(--color-bg-primary) 5%, rgba(221, 194, 185, 0.7) 50%);` + 미세 노이즈 오버레이 (`opacity: 0.1`).
    *   메인 슬로건 (H1): `var(--font-family-script)`를 사용하며, 크기는 화면 중앙 상단에 배치 (`transform: translateY(-20px);`로 시작).
    *   **애니메이션:** 로드 시 슬로건이 아래에서 위로 0.5초간 부드럽게 페이드 인 (Timing: `transition-duration: 0.5s; transition-timing-function: cubic-bezier(0.2, 1, 0.3, 1);`).
    *   **핵심 요소:** 슬로건 아래에 작은 서브텍스트 (`var(--color-text-dark)`).

### 2. OEA 프로세스 설명 박스 (Observe -> Extract -> Automate)
*   **목표:** 핵심 지식을 구조화하여 전달하는 플로우차트 컴포넌트.
*   **규격:** 각 단계별 사각형 블록 (`width: calc(100% - 4rem)`, `height: 25vh`).
*   **상태 변화 (State Change):**
    *   **Default State:** 배경색 `var(--color-bg-secondary)`와 테두리 얇은 차콜 라인. 내부에는 핵심 키워드(e.g., 'Observe')만 표시.
    *   **Active State (Hover/Click):** 마우스 오버 시, 배경색이 미세하게 어두워지며 (`var(--color-bg-secondary)` $\rightarrow$ `rgba(245, 239, 234, 0.8)`) 테두리가 네온 블루 계열로 깜빡이는 듯한 애니메이션 적용 (CSS Keyframe: `neon-glow`).
    *   **Detail Display:** 클릭 시, 박스 아래에 세부 설명 영역이 1초 동안 페이드 인하며 등장 (`transform: translateY(0);` 동작).

### 3. CTA/배너 컴포넌트 (Call To Action)
*   **목표:** 다음 액션 유도 및 수익화 포인트 강조.
*   **규격:** `width: 1080px`, `height: 450px`. 화면의 흐름이 끝나는 지점에 배치.
*   **디자인:** 뉴트럴 어스톤 바탕에 차콜 그레이로 명확한 대비를 이룸. 중앙 하단에 강조 문구("더 깊은 학습이 필요하신가요?") 배치.
*   **애니메이션:** 배경 전체가 좌측에서 우측으로 0.7초간 부드럽게 패닝 인(Panning In)하며, 버튼 영역만 네온 글로우 효과를 주어 클릭 유도.

## 🎯 목적
이 문서는 최종 제작된 모든 시각적 컴포넌트가 다양한 플랫폼과 환경에서 의도한 대로 작동하는지 검증하고, 개발자가 놓칠 수 있는 사소하지만 중요한 디버깅 포인트를 제공합니다.

## ✅ 기술 스펙 점검 항목 (Developer Checklist)
1. **CSS 변수 누락 여부:** 모든 색상 코드 및 크기(padding/margin)가 하드코딩되지 않고 `var(--...)` 형태로 정의되었는가?
2. **타이밍 일관성:** 컴포넌트 A의 페이드 아웃 시간과, 다음 컴포넌트 B의 페이드 인 시간이 겹치거나 너무 길지 않은가? (최적 타이밍: Transition Duration = 0.3s ~ 0.5s)
3. **글리치 효과 연동:** 네온/글리치 노이즈(CSS Keyframes)는 모든 섹션의 전환 지점(Transition Point)에만 적용되었으며, 내용 자체를 가리지 않는가?
4. **모바일 접점 테스트:** 폰트 크기(`var(--font-size-body)`)가 iPhone SE (Small Screen)에서도 최적화되어 가독성을 유지하는가?
