# 📐 최종 비주얼 가이드라인: OEA 시스템 컴포넌트 설계 명세서 (V2.0)

**프로젝트 목표:** 시청자의 '좌절감(Pain)'을 극대화한 후, '구조적 해결책(Solution)'으로 자연스럽게 유도하는 상호작용 경험 제공.
**디자인 철학:** 아날로그적 신뢰성 (뉴트럴 어스톤/빈티지 질감) + 디지털적 명확성 (시스템 컴포넌트 구조).

## 🎨 전역 디자인 시스템 재정의 (Global System Variables)

| 요소 | 변수명 | 값 및 사용처 | 애니메이션 가이드라인 |
| :--- | :--- | :--- | :--- |
| **Primary BG** | `--color-bg-primary` | `#DDC2B9` (뉴트럴 어스톤). 모든 배경에 적용. | 미세 노이즈 레이어 (`opacity: 0.1`)를 기본으로 유지하여 아날로그 감성 부여. |
| **Text Dark** | `--color-text-dark` | `#3A3530`. 본문, 제목 등 주요 정보에 사용. | 진입 시 `translateY(20px)`에서 시작하여 부드럽게 페이드 인 (`opacity: 0` $\rightarrow$ `1`)하며 나타나야 함. |
| **Highlight/Action** | `--color-accent` | `#A87C63`. 강조되는 키워드, 버튼, 플로우 화살표에 사용. | 클릭이나 정보가 활성화될 때만 `scale(1.05)`와 함께 채도 높은 애니메이션을 적용하여 중요성을 부각. |
| **System Font** | `--font-body` | `Pretendard` 또는 유사 산세리프체. | 모든 데이터 블록, 라벨링 텍스트에 사용. 가독성 극대화. |

## ✨ 핵심 단계별 모바일 목업 키프레임 5가지 (Aha! Moment Sequence)

### 🔑 Keyframe 1: Pain Trigger — 좌절감의 시각적 압박
*   **목표:** 스크립트 초반부(`[P_FRUSTRATION]`)에서 사용자의 불안감과 혼란을 극대화하여 몰입도를 확보합니다. (Problem Identification)
*   **비주얼 레이아웃:** 화면 전체가 빠르게 전환되는 정보 콜라주(책, 코드, 강의 영상 썸네일 등)로 가득 차야 합니다. 중앙에는 질문 형태의 핵심 메시지가 배치됩니다.

| 기술 사양서 (Tech Spec) | 상세 내용 |
| :--- | :--- |
| **상태 변화 (State Change)** | 초기 상태는 고정된 배경이 아닌, 무작위 노이즈와 정보 파편(Data Fragments)이 빠르게 깜빡이는 '과부하' 상태여야 함. |
| **전환 애니메이션 (Transition)** | `transition-duration: 0.3s` / `ease-in-out`. 각 데이터 파편은 무작위 속도로 배경을 오가며 시각적 피로도를 유발합니다. |
| **핵심 요소 인터랙션** | 중앙의 질문(`[P_FRUSTRATION]`)이 나올 때, 해당 텍스트만 갑자기 정지하고 (Stutter Effect), 화면 전체의 노이즈가 잠시 사라져서 '진실'을 가리키는 것처럼 연출해야 합니다. |
| **CSS 변수 활용** | `var(--color-bg-primary)` 위에 불규칙한 패턴 애니메이션 레이어를 덧씌움. |

### 🔑 Keyframe 2: System Diagnosis — 문제의 주체 이동 (The Pivot)
*   **목표:** 시선을 개인의 노력에서 '시스템 부재'라는 객관적 구조 문제로 강제로 전환합니다. 이 순간이 첫 번째 지적인 해방감을 제공해야 합니다.
*   **비주얼 레이아웃:** 배경은 안정된 뉴트럴 어스톤으로 돌아오며, 화면 좌측 상단에 굵고 경고성 문구(예: "개인의 의지 부족 $\times$")가 강하게 배치됩니다. 중앙에 핵심 메시지가 명확히 분리되어 나타납니다.

| 기술 사양서 (Tech Spec) | 상세 내용 |
| :--- | :--- |
| **상태 변화 (State Change)** | Keyframe 1의 혼란스러운 배경이 갑자기 `opacity: 0`으로 사라지며, 깨끗하고 정제된 시스템 인터페이스(Sleek UI) 스타일로 전환되어야 합니다. (Dramatic Shift). |
| **전환 애니메이션** | **[핵심 로직]:** Keyframe 1의 노이즈가 마치 '오류 코드'처럼 화면 밖으로 강하게 밀려나가는 효과 (`transform: translateX(-100vw);`)를 사용하여, 문제가 해결되고 새로운 구조로 들어오는 느낌을 강조합니다. |
| **정보 아키텍처** | "문제는 당신의 의지가 부족해서가 아닙니다." 문구 뒤에 `[근거: 시스템적 오류]`라는 작은, 코딩 서체(`var(--font-detail)`) 라벨이 붙으며 전문성을 부여합니다. |

### 🔑 Keyframe 3: Solution Introduction — OEA 모델의 명확한 제시
*   **목표:** 막연했던 해결책을 'OEA'라는 구조화된 프로세스로 포장하여, 지식에 대한 통제감(Sense of Control)을 제공합니다.
*   **비주얼 레이아웃:** 화면 중앙에 3개의 독립적이고 상호 연결된 모듈 박스(Observe, Extract, Automate)가 배치됩니다. 마치 하드웨어 컴포넌트를 조합하는 듯한 느낌입니다.

| 기술 사양서 (Tech Spec) | 상세 내용 |
| :--- | :--- |
| **상태 변화** | OEA 3가지 개념이 순차적으로 등장하며, 각 박스가 활성화될 때마다 주변에 전력선 같은 `glow` 애니메이션 효과가 지나갑니다. |
| **전환 애니메이션 (Transition)** | **[핵심 로직]:** 관찰(Observe) $\rightarrow$ 추출(Extract) $\rightarrow$ 자동화(Automate) 순으로, 마치 데이터 플로우처럼 좌측에서 우측으로 화살표(`SVG Path`)가 움직이며 다음 단계를 예고합니다. `timing-function: cubic-bezier(0.25, 1, 0.5, 1)` (Smooth Acceleration). |
| **상호작용 로직** | 각 OEA 모듈 박스에 마우스를 올리면 (`:hover`), 해당 키워드가 `--color-accent`로 빛나고, 하단에 관련 설명(예: '데이터 변수 분해')이 자동으로 팝업되는 인터랙션 구현. |

### 🔑 Keyframe 4: Deep Dive — 데이터 흐름 시각화 (Flowchart Detail)
*   **목표:** OEA 중 가장 중요한 단계인 '추출(Extract)' 과정을 상세한 플로우차트로 보여주며, 학문적/기술적 깊이를 어필합니다.
*   **비주얼 레이아웃:** 좌측에 입력 데이터(Input Data Stack), 우측으로 진행하는 프로세스 박스 3개, 그리고 최종 결과물(Output)이 명확하게 분리된 다이어그램 형태입니다.

| 기술 사양서 (Tech Spec) | 상세 내용 |
| :--- | :--- |
| **상태 변화** | 데이터가 좌측에서 우측으로 '처리되는 과정'을 시각적으로 보여줘야 합니다. (Input $\rightarrow$ Process 1 $\rightarrow$ Process 2 $\rightarrow$ Output). 이 흐름에 애니메이션이 필수적입니다. |
| **애니메이션 구현** | 각 프로세스 박스는 순서대로 `slide-in`되며, 데이터 변수([VAR])가 들어갈 때마다 해당 위치의 파라미터 값이 점진적으로 채워지는(Typewriter Effect) 효과를 적용합니다. |
| **기술적 강조** | 모든 연결선은 단순한 선이 아니라, 정보의 이동을 의미하는 '데이터 흐름(Data Flow)' 애니메이션으로 표현되어야 합니다. (예: 작은 빛나는 입자들). |

### 🔑 Keyframe 5: CTA/Resolution — 행동 유도 및 상품 노출
*   **목표:** 학습 과정 전체를 성공적으로 마치고, 최종 솔루션([C_OFFERING])을 제시하여 구매 결심을 확정합니다.
*   **비주얼 레이아웃:** 배경이 다시 안정되고 따뜻한 느낌으로 회복됩니다. 중앙에 명확하게 '로드맵' 또는 '클래스 신청 버튼'만 배치되어야 합니다.

| 기술 사양서 (Tech Spec) | 상세 내용 |
| :--- | :--- |
| **상태 변화** | 지금까지의 모든 복잡한 정보(Keyframe 1~4)가 배경으로 빠져나가고, 오직 이 CTA 화면만이 '명료함' 속에서 홀로 존재해야 합니다. (Clarity State). |
| **애니메이션 효과** | 최종 상품 노출 버튼(`[C_OFFERING]`)은 주변의 빛을 모으는 듯한 `pulsing` 애니메이션과 함께, 커서가 올라가면 미세하게 확대되는(Micro-Interaction) 반응이 있어야 합니다. |
| **정보 계층 구조** | 1순위: 상품명 (최대 크기), 2순위: 핵심 이점 (`[T_BENEFIT]`) (중간 크기), 3순위: 구매 버튼 (가장 명확하고 클릭 가능한 영역). |
