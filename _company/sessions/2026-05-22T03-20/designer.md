# 🎨 Designer — Developer가 구현할 프로토타입에 대한 최종 기술 사양서를 기반으로, 첫 번째 영상에 사용할 썸네일 디자인 및 본편 내 모션 그래픽(글리치 효과 포함)의 초기 프로토타입을 생성하여 개발 결과물과 시각적 품질을 즉시 검증하라.

🎨 Designer: 작업 시작합니다. CEO 지시 및 자율 사이클 목표에 따라, 기존의 '네온/글리치' Tech Spec을 기반으로 첫 번째 숏폼 영상의 **썸네일 컨셉 3안**과 본편 내 **모션 그래픽 플로우 초기 프로토타입(Concept Mockup)**을 생성하여 개발팀이 즉시 검증할 수 있도록 산출합니다.

이번 작업은 단순한 '디자인 시안' 제작이 아닙니다. 기존에 정의된 모든 CSS 변수와 애니메이션 타이밍 가이드라인(`transition-duration: 0.5s;`, `ease-out` 등)을 준수하여, **개발자가 코드로 구현했을 때의 최종 결과물 예측 및 검증**을 목표로 합니다.

### 🎨 상세 디자인 스펙 (Concept A: 시스템 에러 - High Tech Impact)

**목표:** 시청자가 '시스템 오류'처럼 느꼈던 시간 낭비의 원인을 명확하고 자극적으로 각인시킨다.
**톤앤매너:** 네온, 디지털, 경고(Warning).

#### 1. 썸네일 스펙 (Thumbnail Spec)
*   **레이아웃:** 화면을 세로로 가르는 **글리치 라인(Glitch Line)**이 배경 전체에 깔림.
*   **컬러 팔레트:**
    *   배경: `var(--color-bg-primary)` (`#DDC2B9`) 위에 글리치 효과를 주기 위해 대비되는 네온 색상 사용.
    *   하이라이트 텍스트: `#00FFFF` (Cyan Neon) / `#FF00FF` (Magenta Neon).
    *   강조 박스: 반투명 블랙 (`rgba(0, 0, 0, 0.6)`).
*   **타이포그래피:** `Pretendard` (굵게), 글리치 효과를 적용하여 마치 화면이 깨지는 것처럼 보이도록 애니메이션화함.
*   **핵심 요소 배치:**
    1.  상단: [경고! 시스템 오류] - 네온 핑크, 작지만 강렬하게.
    2.  중앙 메인 카피: **"당신의 시간은 데이터가 잘못 처리되고 있습니다."** (Cyan Neon) - 최대 사이즈 (`var(--size-h1)`).
    3.  하단: [OEA 프로세스] 로고/심볼 - 작은 글리치 텍스트와 함께 배치하여 해결책을 제시하는 느낌 부여.

#### 2. 모션 그래픽 플로우 (Motion Graphics Flow Spec)
*   **상태 변화:** 정적 이미지 → **시스템 부팅 실패(Boot Fail)** 애니메이션 → 오류 메시지 출력 (`[Error: Time Management Failed]`) → OEA 솔루션을 통한 정상화 과정 시뮬레이션.
*   **애니메이션 상세 지시서 (Codari용):**
    1.  **(0s - 0.5s) Start:** 배경에 미세한 **노이즈 레이어(Opacity: 0.1)**가 빠르게 스캔되는 효과를 주입 (`@keyframes scanLine`).
    2.  **(0.5s - 1.5s) Glitch Trigger:** 메인 카피가 갑자기 수평으로 3~4회 **글리치 번짐(Shifted Pixel Effect)**을 일으키며 나타남. (CSS: `transform: translate(-2px, 0);`와 같은 짧은 변위 애니메이션 반복).
    3.  **(1.5s - 2.5s) Solution Injection:** 글리치 효과가 사라지면서, 화면 중앙에 **'OEA System Active'**라는 녹색 네온 박스가 `fade-in/scale-up`으로 부드럽게 나타남.

### 📐 상세 디자인 스펙 (Concept B: 깨달음의 순간 - Emotional Connection)

**목표:** 시청자가 자신의 경험과 감정을 투영하게 하며, 편안한 공감대를 형성한다.
**톤앤매너:** 빈티지, 따뜻함, 사색적.

#### 1. 썸네일 스펙 (Thumbnail Spec)
*   **레이아웃:** 좌측에 'Pain Point'를 상징하는 손그림/아이콘(시계가 깨진 모습 등), 우측에 희미하게 빛나는 'Solution'을 배치하여 대비 유도.
*   **컬러 팔레트:**
    *   배경: `var(--color-bg-primary)` (뉴트럴 어스톤) - 배경 전체의 70% 비중 차지.
    *   텍스트: `var(--color-text-dark)` (짙은 차콜 그레이).
    *   강조 요소: **세피아 필터**를 입힌 따뜻한 오렌지 계열 (`#E69A8D`)을 사용하여 '깨달음'의 빛을 표현.
*   **타이포그래피:** 상단에 `Brush Script` 또는 유사 캘리그래피체(감성적)로 핵심 슬로건 배치. 본문은 가독성이 좋은 `Pretendard` 사용.
*   **핵심 요소 배치:**
    1.  상단: (Handwritten Style) "이 방법, 왜 이제 알았을까요?" - 감성 연결 유도.
    2.  중앙 메인 카피: **"시간 낭비는 '시스템적 오류' 때문입니다."** - 차분하지만 힘 있는 어조로 전달.
    3.  하단: OEA 프로세스 다이어그램의 간결한 아이콘화 (좌측에 배치).

#### 2. 모션 그래픽 플로우 (Motion Graphics Flow Spec)
*   **상태 변화:** 불안정함(흩어진 이미지들) → 집중된 포커스(하나의 오브젝트가 강조됨) → 명료하게 정렬되는 과정.
*   **애니메이션 상세 지시서 (Codari용):**
    1.  **(0s - 1s) Start:** 화면 전체에 미세한 **필름 그레인 노이즈**를 주입하고, 초점이 흐릿하다가(Gaussian Blur 적용), 중앙의 메인 카피만 서서히 선명해지도록 `transition-duration: 1.2s;` 효과 사용.
    2.  **(1s - 2s) Focus Shift:** 배경에 산재되어 있던 '시간 낭비'를 상징하는 아이콘들(예: 휴대폰, 업무 서류 더미)이 빠르게 회전하다가, 중앙의 OEA 플로우차트 주변으로 **안정적으로 수렴**하며 자리를 잡는 애니메이션 (`ease-out` 기반).

### 📐 상세 디자인 스펙 (Concept C: 데이터 플로우 - Structured System)

**목표:** 내용 자체에 대한 신뢰도를 극대화하고, 지식을 구조화하여 전달하는 학술적 느낌을 준다.
**톤앤매너:** 전문적, 논리적, 깔끔함(Clean UI).

#### 1. 썸네일 스펙 (Thumbnail Spec)
*   **레이아웃:** 3분할 레이아웃 (Pain Point / Process / Solution). 전체를 하나의 **'와이어프레임 박스'** 안에 가두어 웹사이트/시스템 같은 느낌 부여.
*   **컬러 팔레트:**
    *   배경: `var(--color-bg-secondary)` (밝은 크림색) - 깨끗하고 정제된 종이 질감 강조.
    *   강조 요소: 차분한 청회색 (`#6B8E9F`)과 검정(블랙 텍스트 대비).
*   **타이포그래피:** `Courier New` 또는 유사 모노 스페이스체(기술적)를 메인 카피에 사용하고, 설명은 `Pretendard`로 구조화.
*   **핵심 요소 배치:**
    1.  좌측: Pain Point (질문 형태의 텍스트).
    2.  중앙: **[OEA]** 라는 큰 시스템 타이틀과 함께 프로세스 플로우차트가 다이어그램 형태로 자리 잡음. (화살표와 박스로 구성)
    3.  우측: Solution/결론을 상징하는 핵심 데이터 포인트(숫자나 그래프).

#### 2. 모션 그래픽 플로우 (Motion Graphics Flow Spec)
*   **상태 변화:** 질문 제기 → 프로세스 시각화 → 단계별 정보 로딩 완료.
*   **애니메이션 상세 지시서 (Codari용):**
    1.  **(0s - 1s) Start:** 화면 전체가 **'데이터 로딩(Loading Bar)' 애니메이션**을 거치는 것으로 시작. 로딩 바와 함께 '시스템 분석 중...'이라는 메시지를 출력 (`@keyframes loadingPulse`).
    2.  **(1s - 3s) Flowchart Reveal:** 플로우차트의 각 박스와 화살표가 순차적으로 (Sequential) **데이터가 채워지듯** 나타나게 합니다. (예: Box A가 0.5초 등장 → Arrow A→B가 0.3초 등장 → Box B가 0.5초 등장). 이는 개발자가 `setTimeout`이나 단계적 애니메이션을 적용하기 가장 쉬운 구조입니다.

*   **Primary BG:** `var(--color-bg-primary)` (`#DDC2B9`) / 뉴트럴 어스톤

*   **Core Mechanism:** 모든 디자인 요소는 **상태 변화(State Change)**와 애니메이션 타이밍 가이드를 중심으로 정의되어야 함.

## 🎬 모션 그래픽 플로우 최종 통합 가이드
1.  **Hooking (0~3초):** **Concept A**의 '글리치/시스템 에러' 애니메이션을 사용하여 시청자의 주의를 강제로 사로잡는다. (최고 임팩트)
2.  **Problem Identification (3~8초):** **Concept B**의 '빈티지 필름 노이즈 및 포커스 전환' 효과를 활용하여, 문제를 개인적이고 공감 가는 경험으로 정의한다. (공감대 형성)
3.  **Solution/Process (8초 이후):** **Concept C**의 '데이터 플로우(OEA)' 구조와 애니메이션을 사용하여 해결책을 논리적으로 설명하고 신뢰도를 구축한다. (정보 전달)

*   **기술 구현 지침:** 모든 컨셉은 각자의 역할이 명확하므로, 개발팀은 이 3가지 모듈을 순차적인 **상태 변화(State Machine)**로 구현해야 함.
