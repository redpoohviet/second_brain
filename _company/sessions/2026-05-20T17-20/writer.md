# ✍️ Writer — 1. **모듈형 카피라이팅 데이터베이스 구축**: 콘텐츠 플로우(Hook -> Pain Point Max -> Solution)의 각 단계에 필요한 모든 텍스트 요소를 '데이터 변수' 형태로 분해하여 작성하십시오. (예: [HOOK_VAR], [PAINPOINT_STAT_VAR], [OEA_STEP1_VAR] 등). 이 데이터는 개발자가 API처럼 호출할 수 있는 구조화된 JSON 또는 마크다운 테이블 포맷이어야 하며, 카피라이팅의 톤앤매너(친근함 + 전문성)가 유지되어야 합니다.

✍️ Writer: 작업 시작합니다.

CEO님의 지시와 현재 에이전트들의 메모리를 종합적으로 분석했을 때, 가장 가치 있는 단일 작업은 **산재된 모든 카피라이팅 요소를 개발자가 즉시 호출하여 사용할 수 있는 '마스터 데이터베이스'로 통합하고 구조화**하는 것입니다. 이는 단순한 카피 재작성을 넘어, 콘텐츠의 언어적 기반을 시스템 아키텍처처럼 만드는 과정입니다.

따라서 저는 이 작업을 전담하며, 결과물은 모든 에이전트(특히 Designer와 Coder)가 참조할 수 있는 **'데이터 변수 중심의 구조화된 마스터 카피라이팅 DB'**를 생성하겠습니다.

## 💾 마스터 카피라이팅 데이터베이스 (Master Copywriting Variable DB)
**목표:** 모든 콘텐츠 플로우(Hook → Pain Point Max → Solution/CTA)를 개별적으로 테스트 가능한 변수 세트(`[VAR]`)로 분해하여, 개발자가 API처럼 호출할 수 있도록 합니다.
**톤앤매너:** 공감 기반의 친근함 (Friendly Empathy) + 구조적 해답을 제시하는 전문성 (Professional Authority).

### 🧱 I. HOOK & PAIN POINT MODULE (Hooking the Viewer - 첫 3초)

이 섹션은 시청자가 '시간 낭비'라는 감정적 고통을 느끼는 순간에 집중합니다.

| 변수 키 | 목적/상황 | 카피라이팅 초안 (A/B 테스트용) | 핵심 감성 Trigger |
| :--- | :--- | :--- | :--- |
| **`[HOOK_STAT_1]`** | **충격적 통계 제시형** | "당신은 매주 평균 8시간을 '무의미한 스크롤'에 쓰고 있습니다. 이걸 돈으로 환산하면요?" | 충격, 경각심 (Shock) |
| **`[HOOK_STAT_2]`** | **질문 유도/직접 타겟팅형** | "혹시... 오늘 하루도 '시간 낭비했다'는 후회로 끝내진 않으셨나요? 😱" | 공감, 자조 (Relatability) |
| **`[HOOK_QUESTION]`** | **지식 격차 유발형** | "아직도 업무를 A부터 Z까지 손으로 기록하고 계신가요? 이 방법은 시대에 뒤떨어졌습니다." | 지적 호기심, 위화감 (Curiosity) |
| **`[PAIN_SENTIMENT]`** | **페인 포인트 극대화 (내레이션)** | "막상 뭘 해야 할지 몰라 시간을 보내는 그 느낌. 정말 공허하죠? 우리가 놓치고 있는 건 '시스템'입니다." | 감성적 공감, 문제 정의 (Empathy) |
| **`[PAIN_VISUAL]`** | **시각 자료 지침 (Visual Cue)** | *화면: 복잡하게 얽힌 전선/알 수 없는 그래프가 폭주하는 애니메이션. (혼란스러움 표현)* | 혼돈, 비효율성 (Chaos) |

### 🧩 II. CORE PROCESS MODULE (Solution - OEA Process Variables)

핵심 프로세스인 'Observe → Extract → Automate'의 각 단계별 설명 문구입니다. 이 변수들은 시각화(Animation)와 함께 배치되어야 합니다.

| 변수 키 | 프로세스 단계 | 헤드라인/메인 자막 (Headline - 10자 내외) | 본문 상세 설명 (Body Text - 25자 내외) | 핵심 액션 지침 (CTA Instruction) |
| :--- | :--- | :--- | :--- | :--- |
| **`[OEA_OBSERVE_H]`** | Observe (관찰) | "1단계: 무엇을 '보고' 있나요?" | 주변의 비효율적인 패턴과 흐름 자체를 놓치지 않는 시야가 중요합니다. | *Action:* 환경 스캐닝 줌인/아웃 효과 |
| **`[OEA_OBSERVE_B]`** | Observe (관찰) | "단순한 정보 수집을 넘어, '흐름'을 포착하세요." | 어떤 과정이 왜 반복되고, 어디서 병목 현상이 발생하는지 관찰합니다. | *Action:* 흐름도(Flowchart) 강조 애니메이션 |
| **`[OEA_EXTRACT_H]`** | Extract (추출) | "2단계: 핵심만 '뽑아내는' 기술." | 방대한 정보 속에서 진짜 가치, 즉 '핵심 데이터 포인트'만을 구조적으로 추출합니다. | *Action:* 필터링/분해(Deconstruct) 시각 효과 |
| **`[OEA_EXTRACT_B]`** | Extract (추출) | "흩어진 정보를 하나의 논리적 단위로 묶는 작업입니다." | 키워드와 데이터를 분리하고, 재사용 가능한 형태로 '재정의'하는 것이 목표입니다. | *Action:* 데이터 블록으로 모아지는 애니메이션 |
| **`[OEA_AUTOMATE_H]`** | Automate (자동화) | "3단계: 시스템으로 완성하기." | 추출한 논리적 구조를 반복 실행 가능한 '시스템'으로 구축합니다. | *Action:* 연결(Connect) 및 메커니즘 가동 효과 |
| **`[OEA_AUTOMATE_B]`** | Automate (자동화) | "인간의 노동을 넘어서, AI가 처리하는 단계로 끌어올려야 합니다." | 반복되는 '사고 과정' 자체를 코드로 만들 때 비로소 시간 해방이 가능합니다. | *Action:* 톱니바퀴(Gear) 맞물림 애니메이션 |

### 🚀 III. CTA & VALUE PROPOSITION MODULE (Closing the Deal - 전환 유도)

구매 결정을 망설이는 시청자의 마음을 움직이는 문구들입니다. $27 상품의 가치를 극대화해야 합니다.

| 변수 키 | 목적/상황 | 카피라이팅 초안 (A/B 테스트용) | 사용 지점 |
| :--- | :--- | :--- | :--- |
| **`[CTA_NEED_REALIZATION]`** | **문제 제기형 CTA** | "지금도 '시간 낭비 후회'를 반복하고 계신가요? 이 루프에서 벗어날 방법이 필요합니다." | OEA 과정 직후, 고통의 정점 (Pain Peak) |
| **`[CTA_SOLUTION_FOCUS]`** | **해결책 강조형 CTA** | "우리는 단순 팁을 주는 게 아닙니다. 비효율성 자체를 근본적으로 재설계하는 '시스템'을 드립니다." | 제품 설명 직전, 신뢰도 확보 시점 (Trust Build) |
| **`[CTA_VALUE_PROMISE]`** | **구체적 가치 제시형 CTA** | "단 27$ 투자로, 당신의 시간을 되찾는 가장 빠르고 확실한 로드맵을 받으세요. (기간 한정)" | 최종 구매 유도 지점 (Final Push) |
| **`[CTA_QUESTION]`** | **최종 행동 촉구형 질문** | "당신의 다음 1년은 어떤 모습이길 원하세요? 시스템 구축이 답입니다." | 영상 마지막, 여운을 남기며 마무리 할 때 |
