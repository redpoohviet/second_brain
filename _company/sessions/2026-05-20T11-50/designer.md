아래는 모바일 최적화된 (9:16 세로 비율) 두 핵심 에셋에 대한 최종 비주얼 가이드라인 및 레이아웃 브리프입니다.

## 🖼️ [Asset 1] 후킹 타이틀 카드 (Hooking Title Card)
**(활용 시점: 영상 시작 0초 ~ 3초)**
**목표:** 스크롤을 멈추게 할 만큼의 강렬한 경고/충격 메시지 전달. '시간 낭비'라는 감정적 공감대를 즉각적으로 건드린다.

### 1. 디자인 목표 및 컨셉
*   **테마:** 긴급성 (Urgency) & 충격 (Shock).
*   **톤앤매너:** 경고(Warning) 사인 같은 느낌을 필름 질감으로 완화하여 사용합니다.
*   **핵심 구조:** 중앙에 가장 큰 텍스트를 배치하고, 주변 배경은 미니멀하게 유지하여 시선이 오직 메시지에 집중되도록 합니다.

### 2. 비주얼 스펙 및 가이드라인 (Mobile Optimized)
| 요소 | 사양/값 | 상세 지시 사항 |
| :--- | :--- | :--- |
| **사이즈** | 1080px (W) x 1920px (H) (9:16) | 세로 비율 고정. 모든 레이아웃은 이 기준에 맞춤. |
| **배경색** | `#FAF0E6` (Neutral Earthtone) | 전체 배경에 미세한 필름 그레인 텍스처 오버레이 필수. |
| **폰트 계층 구조** | 1순위: Playfair Display Bold/Black | 메시지 본문, 제목 등 가장 강조해야 할 문구에 사용. (고급스러움 + 강렬함) |
| | 2순위: Inter SemiBold | 보조 설명, 부제(Sub-headline) 등 정보 전달 용도에 사용. (가독성 최우선) |
| **강조 색상** | `ACCENT_WARNING`: `#B04D3A` (진한 테라코타 레드/브라운) | 'STOP', '충격' 등의 키워드 및 경고 아이콘에만 사용. |
| **레이아웃 지점** | 1. Top-Third: 부제 (작은 설명). 2. Center Focus: 핵심 문구 (가장 크게). 3. Bottom-Third: CTA/브랜드명. | 메시지 배치 시, 화면의 상하 여백을 확보하여 답답함을 줄입니다. |

### 3. 레이아웃 스케치 상세 지시
1.  **최상단 (Top-Third):** 작은 사이즈로 부제 배치. 예: "혹시 매일 이걸 손으로 하고 계신가요?" (폰트: Inter SemiBold, 크기: 30pt)
2.  **중앙 포커스 (Center Focus):** 가장 큰 문구 배치. 예: **🚨 STOP! 당신의 시간은 돈입니다.** (폰트: Playfair Display Black, 크기: 150~200pt). 이 문구에만 `#B04D3A` 색상 적용.
3.  **배경 처리:** 중앙 포커스 근처 배경을 미세하게 어둡게(Overlay opacity 10%) 하여 시선을 집중시킵니다. (이것이 '충격'의 시각적 표현입니다.)

## 📊 [Asset 2] 프로세스 설명 인포그래픽 (OEA Flowchart)
**(활용 시점: 영상 본론 10초 ~ 23초)**
**목표:** 복잡한 개념(OEA)을 단계별 플로우차트로 명료하게 전달. 지적이고 구조적인 느낌을 주어 권위를 확립한다.

### 1. 디자인 목표 및 컨셉
*   **테마:** 시스템화 (Systematization) & 흐름 (Flow).
*   **톤앤매너:** 깔끔하고 정돈된 다이어그램 스타일. 빈티지한 배경 질감 속에서 기술적인 명확성을 유지합니다.
*   **핵심 구조:** 3개의 박스(Box)가 순차적으로 연결되고, 각 박스는 해당 프로세스의 핵심 키워드를 시각화해야 합니다.

### 2. 비주얼 스펙 및 가이드라인 (Mobile Optimized)
| 요소 | 사양/값 | 상세 지시 사항 |
| :--- | :--- | :--- |
| **사이즈** | 1080px (W) x 1920px (H) (9:16) | 세로 비율 고정. 플로우차트가 위에서 아래로 흐르도록 설계합니다. |
| **배경색** | `#FAF0E6` + 필름 그레인 | 배경은 미니멀하게 유지하되, 섹션 경계에만 은은한 톤 변화를 줍니다. |
| **폰트 계층 구조** | 1순위: Playfair Display Bold/Black | 각 단계의 제목 (Observation, Extraction, Automation)에 사용. |
| | 2순위: Inter Regular | 설명 문구(Definition), 세부 내용에 사용. |
| **색상 시스템** | `PRIMARY_BOX`: `#A8B69F` (뮤트 Sage Green) | 모든 단계 박스의 기본 배경색으로 사용하여 '기술적 안정감'을 부여합니다. |
| | `ACCENT_FLOW`: `#C37D5F` (테라코타/브라운) | 화살표, 연결선, 최종 결과물(Output Data)에 사용. 흐름과 중요도를 강조합니다. |

### 3. 레이아웃 스케치 상세 지시
1.  **구조:** Top-to-Bottom 순차적 플로우 (Vertical Stack).
2.  **섹션 구성:** 각 단계는 다음의 계층 구조를 가집니다.
    *   (Title Box) -> [키워드: Observation]
    *   (Flow Arrow) -> ⬇️
    *   (Content Area) -> [설명 및 예시 이미지/아이콘]

3.  **애니메이션 브리프 (코다리 핸드오버용):**
    *   **전체 동작:** 스크롤 기반으로 섹션이 나타날 때, 각 박스가 바닥에서 떠오르듯(Fade-up) 순차적으로 등장해야 합니다.
    *   **O → E 연결 시점:** Observation Box가 70% 정도 화면에 진입할 때, `ACCENT_FLOW` 화살표가 먼저 그려지고 (Draw Animation), 그 뒤에 Extraction Box 전체가 나타나야 합니다.
