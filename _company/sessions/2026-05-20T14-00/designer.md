### ✨ Template Set 1: [THE HOOK] 문제 제기 & 충격 경고 (Observation)
**목표:** 영상 초반 3초 내 시청자의 '정보 결핍' 또는 '후회 감정'을 자극하여 리텐션을 확보하는 타이틀 카드 및 후킹 섹션.
**API 연동 요소:** 없음. 오직 강력한 메시지 전달에 집중.

| 컴포넌트 | 규격 (px) | 내용/역할 | 애니메이션 & 기술 스펙 |
| :--- | :--- | :--- | :--- |
| **헤드라인 영역 (A)** | 120px 높이, 전폭 | 핵심 질문 또는 충격적 통계 (예: "당신도 매일 시간을 버리고 있진 않나요?") | **[Animation]** `opacity: 0` $\rightarrow$ `opacity: 1` (Fade-in). **[Timing]** 영상 시작 후 $0.2s$에 애니메이션 시작, $0.5s$ 동안 진행되어 강한 인상 남기기. |
| **서브 메시지 영역 (B)** | 60px 높이, 전폭 | 핵심 키워드 반복 노출 (예: "시간 낭비" / "비효율적 습관") | **[Animation]** 타이핑 효과(Typing Effect) 또는 좌우 스케일 인($\text{scale}(1)$). **[Timing]** $0.8s$ 간격으로 키워드를 변경 노출하며 긴장감 유지. |
| **배경 패턴 (C)** | 전폭 | 뉴트럴 어스톤 배경에 미세한 빈티지 필름 그레인 오버레이. | **[Effect]** 배경 전체에 $0.5s$ 동안 느린 펄싱(Pulsing) 효과를 주어 영상에 깊이감 부여. (JS로 `mousemove` 감지에 따라 아주 미묘하게 색상 변화). |

### ✨ Template Set 2: [THE PROCESS] OEA 플로우 인포그래픽 (Extraction & Automation)
**목표:** '관찰 $\rightarrow$ 추출 $\rightarrow$ 자동화' 과정을 시각적으로 이해시키고, **실시간 API 데이터를 Placeholder로 연동하여 신뢰도를 극대화**하는 핵심 섹션.
**API 연동 요소:** 필수. 단계별 데이터 로딩 및 전환 애니메이션 구현 필요.

| 컴포넌트 | 규격 (px) | 내용/역할 | 애니메이션 & 기술 스펙 |
| :--- | :--- | :--- | :--- |
| **OEA 스테이지 박스** | 3개 분리 배치, $200\text{px}$ 너비씩 | 각각 Observe, Extract, Automate 레이블. 각 단계별 핵심 원리를 시각화. | **[Animation]** 스크롤 기반으로 진입(Intersection Observer)했을 때만 `scale(1)`로 등장하며 부드럽게 팝업. **[Transition]** 다음 스테이지로 넘어갈 때, 현재 박스가 축소되며 배경이 변하는 전환 효과 (CSS Transition: $\text{transform: scale}(0.8)$). |
| **API 데이터 Placeholder ($\text{data-api}$)** | $150\text{px} \times 100\text{px}$ 크기 (각 단계별 배치) | 실제 API가 연동되어 받아오는 데이터를 표시하는 영역. (예: "당신의 비효율 시간: $\mathbf{4h 32m}$"). | **[Animation]** 데이터 로딩 시, 기본적으로 투명한 오버레이(`--color-placeholder`)로 가려져 있다가, API 호출 성공 $0.5s$ 후 부드럽게 `opacity: 1` 되며 글자가 나타나야 함 (Fade-in + Typewriter effect). **[Tech Spec]** 이 영역에는 반드시 $\text{data-api}$ 속성이 부여되어 JavaScript 이벤트 리스너가 연결되어야 합니다. |
| **전환 흐름선 ($\rightarrow$)** | 각 박스를 이어주는 화살표/라인. | 시각적인 프로세스의 연속성을 부여. | **[Animation]** 이전 단계의 데이터 로딩이 완료된 후, $0.3s$ 간격으로 선 자체가 '그려지듯' (Stroke Dashoffset) 나타나야 합니다. |

### ✨ Template Set 3: [THE CTA] 솔루션 제시 및 구매 유도
**목표:** 콘텐츠의 핵심 가치를 요약하고, 시청자의 즉각적인 행동(구매/등록)을 유도하는 마무리 화면. (가장 높은 전환율이 필요한 지점).
**API 연동 요소:** 결제 프로세스 혹은 다음 단계 등록 버튼 데이터 바인딩.

| 컴포넌트 | 규격 (px) | 내용/역할 | 애니메이션 & 기술 스펙 |
| :--- | :--- | :--- | :--- |
| **핵심 요약 헤드라인** | $200\text{px}$ 높이, 전폭 | "OEA 프로세스로 비효율성을 끝내세요." (최종 결론 제시). | **[Animation]** 화면에 나타나기 직전까지 배경을 어두운 오버레이로 처리했다가($\text{opacity: 0.8}$), $0.2s$ 간격으로 밝아지며(`opacity: 1`) 강조하는 극적인 연출 필요. |
| **가격/상품 정보 박스** | 중앙 배치, 가변 크기 | 상품 이름, 가격(예: \$27), 핵심 혜택 리스트 (3가지). | **[Animation]** `pop-in` 효과 사용. 화면 아래에서 위로 $0.4s$ 동안 부드럽게 올라오며 등장해야 합니다. 정보를 한 번에 폭발적으로 보여주지 않도록 주의합니다. |
| **Call To Action 버튼 ($\text{btn-cta}$)** | 전폭, 고정 배치 | "지금 시작하기" 또는 "웨비나 등록". (가장 중요) | **[Animation]** 버튼은 항상 `opacity: 1`로 유지하되, 마우스 오버/터치 시 애니메이션을 적용해야 함. (예: 배경색이 $0.2s$ 동안 약간 밝아지며 미세한 스케일 업($\text{scale}(1.05)$) 효과). **[Tech Spec]** 이 버튼은 클릭(또는 터치) 이벤트 발생 시, 외부 랜딩 페이지 URL로 즉시 리다이렉트되어야 합니다. |
