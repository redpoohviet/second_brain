## ⚙️ II. Data Analysis Framework (분석 프레임워크)

본 보고서의 분석은 다음 세 가지 핵심 데이터를 기준으로 합니다.

1.  **CTR 데이터**: 어떤 비주얼 컴포넌트가 클릭을 유도했는지 (클릭 발생 지점/유형).
2.  **Retention Data**: 사용자가 어느 구간에서 이탈했는지 (이탈률 급증 시점).
3.  **Feedback Data**: 사용자 코멘트를 통해 발견된 시스템적 불편 요소(Pain Point).

| 컴포넌트 | 주요 역할 | 측정 지표 | 현재 성과 (가정) | 개선 필요성 |
| :---: | :---: | :---: | :---: | :---: |
| **Hooking Title Card** | 시선 사로잡기, 주제 설정 | 노출 대비 평균 스킵률 감소율 (↓), 다음 영상 클릭 유도율 | 높음. 하지만 텍스트가 너무 작고 정적임. | 상호작용성 강화 및 가독성 증대. |
| **OEA Process Flow** | 정보 전달, 시스템 구조 설명 | 사용자가 끝까지 시청한 비율 (Retention Rate) | 보통. 과정은 이해하지만 지루함을 느낌. | 애니메이션 타이밍 변화를 주고 '결과'의 중요성을 강조해야 함. |
| **Final CTA Screen** | 행동 유도 (구매/구독), 전환 | 최종 액션 버튼 클릭률 (CTR), 구매 경로 진입율 | 낮음. 마지막 순간에 정보 과부하가 발생함. | 간결성 극대화 및 단일 목적에 집중해야 함. |

## 🎨 III. Component-Specific Optimization Specs (컴포넌트별 상세 개선 사양)

### 1. Hooking Title Card (후킹 타이틀 카드)
*   **[문제 정의]**: 현재는 '정보 제시'에만 머물러 있어, 사용자의 감성적 반응을 유도하는 힘이 약합니다. [근거: Designer 개인 메모리 / 지난 의사결정 로그]
*   **[개선 목표]**: "후회"와 "해결책 발견"이라는 대비되는 감정을 3초 이내에 시각적으로 폭발시키기.
*   **[A/B 테스트 가설 및 사양]**:
    *   **가설 A (현재):** 정적인 손글씨체 + 뉴트럴 어스톤 배경.
    *   **가설 B (개선안 - 권장):** **[애니메이션 강화]** 0~1초에 '충격적 사실' 문구가 빠른 속도로 플래시(Flash) 효과를 주며 등장하고, 이후 텍스트가 마치 필름 노이즈를 거치며 부드럽게 나타남.
    *   **Tech Spec**: `transition-duration: 0.2s;` (플래시), `opacity: 0 -> 1` (부드러운 등장).

### 2. OEA Process Flow (관찰→추출→자동화 흐름도)
*   **[문제 정의]**: 각 단계의 설명이 분리되어 있어, 세 단계가 유기적으로 연결된다는 '시스템'으로서의 느낌이 약합니다.
*   **[개선 목표]**: 각 단계가 마치 **하나의 기계장치(Machine)**처럼 순차적이고 필연적으로 움직이는 느낌을 부여하여 지루함을 해소한다.
*   **[A/B 테스트 가설 및 사양]**:
    *   **가설 A (현재):** 박스형 컴포넌트 분할 배치.
    *   **가설 B (개선안 - 권장):** **[상호작용 로직 강화]** '관찰' 단계 완료 시, 배경 전체에 미세한 전력 흐름(Electricity flow) 애니메이션이 발생하며 다음 단계로의 전환을 암시. 각 박스는 단순히 정보만 담는 것이 아니라, 데이터가 *흐르는 파이프* 형태를 띠도록 디자인한다.
    *   **Tech Spec**: `on('Observe Complete'): trigger_background_glow(duration: 0.7s);` (JS Event Listener 필수).

### 3. Final CTA Screen (최종 행동 유도 화면)
*   **[문제 정의]**: 너무 많은 정보(구매 링크, 구독 버튼, 다음 영상 추천 등)를 한 번에 보여줘 사용자가 무엇을 해야 할지 혼란스러워합니다. [근거: Designer 개인 메모리]
*   **[개선 목표]**: 오직 하나의 액션 (Single Call to Action)만 남기고 모든 시각적 노이즈를 제거한다.
*   **[A/B 테스트 가설 및 사양]**:
    *   **가설 A (현재):** 여러 버튼과 텍스트 블록 배치.
    *   **가설 B (개선안 - 권장):** **[정보 최소화]** 화면 중앙에 가장 강력한 단일 CTA 문구만 남기고, 배경은 극도로 미니멀하게 처리한다. 이 문구가 '시스템 오류 해결'의 마지막 퍼즐 조각처럼 느껴지도록 디자인하여 즉각적인 행동을 유도한다.
    *   **Tech Spec**: `var(--cta-button-size)`를 최대화하고, 버튼 주변에 초점(Focus) 효과 애니메이션을 적용하여 시선이 분산되지 않게 한다.
