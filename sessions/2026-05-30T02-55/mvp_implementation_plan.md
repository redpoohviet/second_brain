## 1. 핵심 전략 및 목표

### 🎯 MVP 의 존재 이유
본 프로젝트는 **'시간 낭비에 대한 후회(Pain Point)'**를 해결하는 솔루션으로, 사용자 여정에서 **T=12~15 초** 지점에서 가장 큰 감성적 고통을 자극한 직후에 저가 진입 상품 ($27) 을 노출하여 구매 전환율을 극대화하는 것이 핵심 목표입니다.

### 🎯 핵심 KPI 스펙 (MVP 출시 전 필수 달성 조건)
| 지표명 | 목표 수치 | 측정 포인트 및 정의 |
| :--- | :--- | :--- |
| **CTA 클릭률 (CTR)** | **4% 이상** | Pain Point Trigger 구간(12~15초) 내 노출된 $27 버튼의 클릭 대비 노출 비율. |
| **이탈율 감소율** | **기존 대비 15%** | '관찰(Observation)' 단계 종료 시점부터 CTA 직전까지의 이탈 방지 (Retention Boost). |
| **평균 시청 지속 시간** | **22~26 초** | 30 초 영상 기준. 사용자가 CTA 클릭 직전까지 콘텐츠에 몰입하는지 측정. |

> **💡 전략적 원칙:** 판매하는 것은 '지식'이 아니라, **'시스템 오류를 해결할 수 있는 통제권(Control)'**입니다. 초기에는 $27 진입 장벽으로 트래픽 확보 후, 구독자 5,000 명 도달 시 Tier B/C 번들로 AOV 극대화 구조를 가집니다.

## 2. 기술 구현 가이드 (Tech Spec)

### 🛠️ 스택 및 환경
- **기술:** HTML/CSS/JS (Vanilla JS) 또는 단일 파일 React (초기 버전은 Vanilla 권장). 복잡한 프레임워크 불필요.
- **배포:** 정적 호스팅 (GitHub Pages, Netlify 등) 가능하도록 경량화.
- **피드백 수집:** 간단한 이벤트 로그 (클릭, 스크롤, 체류 시간) 기록용 `console.log` 또는 `localStorage` 기반.

### 🧩 핵심 컴포넌트 및 기능 요구사항

#### 1. 시각적 에셋 (Designer 제공 기준)
- **Before/After 이미지:** 학습 전(혼란스러운 상태) vs 후(명료해진 시스템) 비교 이미지 배치.
- **OEA 그래프:** 사용자 효율성 향상 과정을 시각화한 차트 (CSS/Canvas 또는 SVG 권장).
- **Knowledge Checkpoint:** 학습 단계별 체크포인트 UI (Progress Bar, Completion Mark).

#### 2. CTA 트리거 로직 (Flowmap 반영)
- **Trigger Point T=12~15s:** 영상 재생 시점 12 초에서 15 초 사이, 사용자의 감성적 고통이 최고조에 달할 때 $27 진입 배너 노출.
- **CTA 버튼 텍스트:** **"더 이상 '느낌'에 의존하지 마세요. 시스템으로 증명하세요."** 또는 **"단 12개월로 시간 벌기 ($27)"**
- **버튼 디자인:** 저항감이 낮은 컬러 (예: 주황색/청록색), 명확한 여백 확보.

#### 3. 마이크로 퍼널 로직
- **Observation Stage:** 사용자가 '관찰' 단계에서 자신의 비효율성을 인지하는 지점까지 이탈 방지 (Loading State 최소화, 자동 스크롤 유도).
- **Conversion Stage:** $27 상품 구매 버튼 클릭 시 즉시 결제 페이지로 이동 (중간 단계 생략).
