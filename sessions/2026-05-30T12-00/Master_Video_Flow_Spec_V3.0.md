# 📺 Jinny`s Lab: 마스터 비디오 플로우 기술 명세서 (V3.0)

## 🎯 1. 프로젝트 개요 및 목표
*   **콘텐츠 형식:** 유튜브 쇼츠 / 인트로 카드 (Short-form Video)
*   **주제:** 시스템 안정화 기반 자동화 효율성 극대화 ('시스템이 나를 대신해 일할 때')
*   **KPI 연동 핵심 메시지:** 시간 낭비 후회하지 마세요. 비즈니스 효율성은 수동 작업 대비 최소 3배 이상 증가합니다. [근거: Writer]
*   **디자인 목표:** 정적인 이미지가 아닌, 사용자의 '상태 변화(State Change)'를 통해 정보의 흐름과 신뢰도를 극대화하는 인터랙티브 시스템 컴포넌트 구현. [근거: Designer 개인 메모리]

## 🎨 2. 비주얼 스타일 가이드 재확인
*   **컬러 팔레트:** Primary BG (`#DDC2B9`), Text Dark (`#3A3530`), Secondary BG (`#F5EFEA`). (CSS 변수 사용 필수) [근거: Designer 검증된 지식]
*   **폰트 시스템:** Hooking Title Card는 캘리그라피체(감성적), 본문/정보 전달은 산세리프체(기술적). [근거: Designer 검증된 지식]
*   **질감:** 뉴트럴 어스톤 기반 + 빈티지 필름 노이즈 레이어 (Opacity 0.1) 필수 적용.

## 🎬 3. 시간대별 인터랙티브 플로우 상세 명세 (Flowchart & Tech Spec)

### A. [Timecode: 0s - 3s] ⚡️ Hooking Phase (문제 제기 및 시선 강탈)
*   **Writer 스크립트:** "아침에 일어나서도 월급이 들어오나요?" / "왜 여전히 수동으로 일하고 있나요?"
*   **시각 컴포넌트:** **'Pain Point Trigger Card'** (클릭/스크롤 반응형 카드)
*   **레이아웃 명세:**
    1.  화면 전체에 배경 노이즈 레이어 적용. [CSS: background-image: radial-gradient(circle, rgba(255, 255, 255, 0.08) 1px, transparent 0);]
    2.  중앙에 메인 질문 타이포그래피 배치. `var(--size-h1)` 크기.
    3.  **애니메이션 로직:** 화면이 **'빠른 페이드 인(Fast Fade In)'**으로 시작하며, 배경에 미세한 깜빡임 효과(`opacity: 0.2` -> `opacity: 1`)를 주어 불안정성을 암시. [Transition-duration: 0.3s;]
    4.  **상호작용 로직:** (가상) 사용자가 시청할 때, 질문 문구 주변에 미세한 **'글리치(Glitch)' 효과** 애니메이션을 적용하여 '시스템 오류'의 느낌을 강조해야 함. [CSS/JS: @keyframes glitch { ... }]

### B. [Timecode: 3s - 15s] 💡 Solution Introduction (핵심 개념 도입)
*   **Writer 스크립트:** "시스템 오류를 만드신 건 당신, 아니라 시스템입니다. 제가 만들어준 자동화 시스템이 당신의 업무를 대신합니다." / "관찰 → 추출 → 자동화"
*   **시각 컴포넌트:** **'OEA 프로세스 플로우차트 인터랙션 모듈'** (Flowchart Component)
*   **레이아웃 명세:**
    1.  Pain Point Card가 빠르게 사라지며, 배경이 **Secondary BG (`#F5EFEA`)**로 전환됨. [Transition-duration: 0.8s;]
    2.  화면 중앙에 OEA 프로세스 다이어그램을 좌우 구조 분할 방식으로 배치 (좌: 관찰/문제점, 우: 추출/자동화). [근거: Designer 개인 메모리]
    3.  **애니메이션 로직:** '관찰' 모듈이 먼저 **Fade In + Slide Up**으로 등장하고, 이후 데이터가 처리되는 느낌을 주며 '추출', 그리고 가장 밝게 빛나는 듯한 효과와 함께 '자동화'가 마지막에 완성되도록 타이밍을 설계. [Timing: 0s (관찰) -> 1.5s (추출) -> 3s (자동화)]
    4.  **상호작용 로직:** 각 단계(O, E, A)마다 **데이터 처리 애니메이션**을 구현해야 함. (예: 데이터 블록이 왼쪽에서 오른쪽으로 이동하며 연결되는 시각적 효과).

### C. [Timecode: 15s - 40s] ✨ Value Proposition (가치 증명 및 CTA 유도)
*   **Writer 스크립트:** "자동화 시스템이 당신의 업무를 처리하면, 당신은 더 중요한 일에 집중할 수 있습니다." / "시간을 되찾으세요"
*   **시각 컴포넌트:** **'Before & After 비교 시퀀스'** (Side-by-Side Comparison Module)
*   **레이아웃 명세:**
    1.  화면이 다시 **Primary BG (`#DDC2B9`)** 톤으로 회귀하며, 좌우 분할 구조가 유지됨.
    2.  **좌측 (Before):** 수많은 손글씨/복잡한 데이터 레이블로 가득 찬 이미지를 배치하고, 낮은 대비와 혼란스러운 애니메이션(Shake Effect)을 부여.
    3.  **우측 (After):** 깔끔하게 정리된 UI 컴포넌트 템플릿을 배치하며, 높은 대비와 부드러운 그리드 레이아웃(Grid Layout)을 유지.
    4.  **전환 애니메이션:** 좌측이 서서히 '흐려지며(Blur Out)', 우측의 깔끔한 정보가 마치 **'정리되는 마법'처럼 스케일 업하며 등장**해야 합니다. [Transition-duration: 1s; ease-out.]

## 🛠️ 4. 코딩/개발팀 전달 지침 (Developer Notes)
*   이 명세서의 모든 애니메이션 타이밍과 좌표는 CSS 변수 기반으로 정의되어야 합니다. (예: `var(--size-h2)`, `var(--timing-transition)`).
*   **[필수 요청]**: 이 플로우를 실제로 모바일 목업 비디오로 구현할 때, **'Aha! Moment'**가 발생하는 시점(OEA 프로세스 완성 및 Before/After 전환)에서 인터랙션 효과(예: 사운드 이펙트 동기화 또는 강한 화면 흔들림)의 타이밍을 명시해 주십시오.
